CS605 教材线性精讲：从“字符串集合”学到 NP-complete

CS605 不是在考背定义，而是在考你能不能判断一个问题属于哪类，并用合适的证明工具说清楚。整门课可以沿着一条线学：先把问题写成语言，再看什么机器能处理它，再证明机器不够强，最后讨论算法快不快。

字符串和语言 FA / PDA TM / Decidable HALT 归约 P / NP 考试台阶

0. 把题目翻译成“哪些字符串算 yes”

理论计算机不直接讨论“这个 Java 程序好不好”，而是先把每个问题变成一个 yes/no 问题。输入是一个字符串；如果字符串满足题目条件，它就在语言里；不满足，就不在语言里。

核心翻译：Language L = all encodings of inputs for which the answer is yes.

例如 L = {<M,w> : M halts on input w} 的意思是：把机器 M 和输入 w 编码成字符串；如果 M 在 w 上会停机，这个编码就在语言里。

先把符号降到地面

Alphabet Σ允许使用的字符集合。像键盘上被允许出现的字符清单。{0,1} 表示只能用 0 和 1。
String / word字符排成的一串。010、abba 都是 string。空串写作 ε。
Σ*alphabet 上所有有限字符串的集合，包括空串。它不是乘号，而是“所有可能长度”的意思。
Language一组字符串。题目里的条件就是筛选规则：满足条件的字符串进入集合。
Encoding把机器、图、公式、程序等对象写成字符串。理论课需要这样做，因为 TM 的输入必须是字符串。
Complement同一个 universe 里所有不在 L 的字符串。证明 TR / co-TR 时常用。

读语言定义的四步

  1. 先找输入长什么样。如果有尖括号 <M,w>，输入通常是“某个对象的编码”。如果是 {a^n b^n : n ≥ 0}，输入就是普通字符串。
  2. 再找 yes 条件。冒号或竖线后面的条件就是加入语言的规则，例如“halts”、“accepts”、“contains a clique”。
  3. 把条件改写成人话。不要一看到公式就证明。先说“这个语言收集的是所有会停机的程序输入对”。
  4. 判断该用哪类工具。普通字符串结构常接 pumping lemma；机器行为常接 decidable / recognisable / HALT reduction；图或集合优化常接 NP。

过关小题：看到 L={<G,k> : G has a clique of size at least k}，先不要写 NP。先说输入是图 G 和数字 k 的编码，yes 条件是图里存在至少 k 个两两相连的点。

内容理解题（先做 3 题）

核心答案（先确认概念）：Language 是一组字符串。题目条件决定哪些字符串是 yes instance，满足条件的字符串就在语言里。

为什么这是对的：理论计算机科学里，"问题"和"语言"是同一枚硬币的两面。你不直接问"这个程序会停机吗"，而是问"<M,w> 这个编码在不在 HALT 语言里"。这样就把所有计算问题统一成了集合成员判断——一个字符串要么在集合里（yes），要么不在（no）。这种统一让你可以用同一套数学工具（自动机、归约、复杂度类）去分析看似完全不同的题目。

题目语境怎么用：看到任何 CS605 大题，第一件事不是翻证明模板，而是把题面翻译成语言定义：输入长什么样？yes 条件是什么？比如"Java 程序在某个时刻 a+b=c"这个题，输入是 <J,a,b,c> 的编码，yes 条件是存在某个运行时刻满足等式。这一步翻译对了，后面选 pumping、decider、reduction 还是 NP verifier 就自然清楚了。

卷面写法：做题时先在草稿纸上写下 L = { <...> : ... } 的完整形式，把冒号前的输入编码形式和冒号后的 yes 条件都写清楚。这一步写对了，整个证明的方向就不会歪。

常见失分点：很多同学一看到编码题就直接套 HALT reduction，但题面可能只是在考你把问题写成语言的能力。如果语言定义都写错了，后面的证明再漂亮也没分。先判断输入类型再选工具。

核心答案（先确认概念）：因为理论模型里的 TM 输入是字符串；编码让程序、图、公式都能作为语言成员被讨论。

为什么这是对的：Turing Machine 的纸带上只能写符号，输入必须是有限字符串。但计算对象——程序、图、公式——天然不是字符串。编码就是桥梁：你给每个对象分配一个唯一字符串，比如用邻接矩阵表示图、用 ASCII 表示程序源码。这样 TM 就能"读"这些对象了。更深一层说，编码让"关于程序的判断"变成"关于字符串的判断"，于是可以用语言理论统一处理所有计算问题。

题目语境怎么用：大题里看到 <M>、<G,k>、<φ> 这种尖括号，就是在告诉你输入是编码后的对象。你不需要真的写出编码细节，但要理解"输入是一个字符串，这个字符串代表某个对象"这层关系——它决定了你是用字符串结构工具（pumping）还是程序行为工具（TM/归约）。

卷面写法：定义语言时写清楚 <...> 里是什么对象的编码，比如 <G,k> 表示图 G 和整数 k 的编码。不要写"输入是图 G"，要写"输入是 <G> 即 G 的编码"。这会影响后续复杂度分析——输入大小是编码长度，不是图中顶点数本身。

常见失分点：把编码等同于原对象。虽然日常说"输入是图 G"，但在形式定义里要说"输入是 <G> 即 G 的编码"。这个概念会影响到复杂度分析——输入 size 是编码长度，不是图中顶点个数本身。

核心答案（先确认概念）：先找输入长什么样，再找 yes 条件，最后判断该用 pumping、decider、reduction 还是 NP 工具。

为什么这是对的：语言定义是整道题的"根"。输入的形式直接决定了你的证明工具选择：如果输入是纯字符串（如 a^n b^n），你走 pumping lemma 路线；如果输入是机器或程序的编码（如 <M,w>），你走识别/可判定/归约路线；如果输入是图或集合加数字（如 <G,k>），你走 NP 路线。不看输入格式就套模板，等于连题都没读懂。

题目语境怎么用：拿到题先圈出输入部分，判断它是"裸字符串"还是"对象编码"还是"图/集合结构"。裸字符串 → pumping；机器编码 → TM/归约；图结构 → NP。这个分类在考试中正确率最高。

卷面写法：第一步永远是"Let us first understand the language: the input is ... and the yes condition is ..."。一句翻译能帮你和阅卷人都锁定方向。

常见失分点：把 yes 条件和输入格式搞混。比如 {a^n b^n} 的 yes 条件是 n 个 a 后跟 n 个 b，"a 后面跟 b"是格式、"数量相等"才是条件。格式和条件分离不清会导致后面 pumping witness 选错。

考试题目题（再做 5 题）

标准答案（先写结论）：输入是用 # 分隔的偶数个 unary blocks，例如 a#aa#...；yes 条件是某个奇数位置 block 等于下一个 block。

为什么这是对的：L1A 定义里，wi∈{a}* 表示每个 block 只由 a 组成（unary 编码，长度就是数量）。# 是分隔符。下标 w2n 说明 block 总数必须是偶数——这很重要，因为后面要找奇数位置 i 去和 i+1 比较。冒号后条件"存在某个奇数 i 使 wi=wi+1"，只要有一对相邻的奇数-偶数 block 等长就算 yes，不是所有对都要相等。

题目语境怎么用：这道题属于 Q1（两类 pumping）的基础小题，先确认语言定义。后面 (b) 要求证 not regular，(c) 可能证 not CFL。只有把"偶数个 block"和"存在某个奇数位置相等"吃透了，pumping 时才知道要破坏什么。

卷面写法：用一句英文："The input is an even-length sequence of unary blocks separated by #. It is a yes instance if there exists an odd index i where block i and block i+1 have the same length."

常见失分点：漏掉"偶数个 blocks"这个条件，或者把 wi=wi+1 理解成"所有相邻 block 对都相等"而忘了是"存在某个奇数 i"。

标准答案（先写结论）：是在问 Java program 的运行行为，所以更接近 recogniser/模拟，而不是 pumping lemma。

为什么这是对的：这个语言的输入是 Java 程序 J 和变量名 a,b,c 的编码——尖括号已经告诉你这不是裸字符串。yes 条件涉及 J 的运行过程："存在某个时刻 a+b=c"。这意味着判断 yes/no 需要执行或模拟 J，不能只看字符串的静态结构。这就是区分 pumping 题和 TM 题的关键线索：输入里有程序，条件涉及运行时行为——走向构造 recogniser 或归约。

题目语境怎么用：这种"程序运行时 eventually 发生某件事"的题，标准做法是构造一个 recogniser 模拟 J 的执行，一旦检测到 a+b=c 就 accept。由于 J 可能永远不满足条件（无限循环），recogniser 在 no instance 上可能永远跑下去——这正是 Turing-recognisable 的典型形状。

卷面写法：先点明"这是程序行为题而非字符串结构题"，然后给出 simulator 的描述：模拟 J 的 step-by-step execution，每一步检查变量值，满足条件则 accept。承认 no instance 上可能不终止。

常见失分点：误以为可以判定。如果 J 本身不停机且永远不满足 a+b=c，没有任何通用算法能在有限时间内确认这一点——所以它不是 decidable，只是 recognisable。

标准答案（先写结论）：输入是 graph G,k；yes 条件是存在 size k 的学生集合，集合中任意两人都是朋友。

为什么这是对的：这道题把抽象图论概念落到具体场景：学生是 vertices，朋友关系是 edges。yes instance 就是存在 k 个学生两两认识——这是经典的 clique 问题。"两两认识"= "任意两个选中学生之间都有边" = "选中顶点构成完全子图"。一旦建立这个对应，后面证明 in NP 和 NP-complete 就有了明确目标。

题目语境怎么用：这道题的语境是 NP/NP-complete。先在 Q6A 证明 in NP（给 certificate 和多项式 verifier），然后 Q7 从 3-SAT 归约证明 NP-complete。yes 条件定义决定了 certificate 长什么样：一个 k 大小的学生集合。

卷面写法：输入是 <G,k>，G 是学生社交图，k 是目标 clique 大小。yes 条件：∃S⊆V, |S|=k, ∀u,v∈S, u≠v → (u,v)∈E。

常见失分点：把"任意两人"写成"至少两人"，或者遗漏 clique size 恰好是 k。"两两认识"要求每一对都有边，不是"每个人都认识至少一个其他人"。

标准答案（先写结论）：把目标 line 放在模拟 M(w) 之后；模拟停机才执行该 line，不停机则永远不到。

为什么这是对的：这是经典的 HALT 归约"后置触发"技巧。给任意 <M,w>，构造新程序 N：N 忽略自己的输入，开始模拟 M 在 w 上的运行。如果 M(w) 停机，N 执行目标行；如果不停机，N 的模拟也永远跑不完，目标行永远不会被触及。于是"目标行是否执行"= "M(w) 是否停机"。如果存在 decider 能判断任意程序的目标行是否执行，套在 N 上就能判定 HALT——矛盾。

题目语境怎么用：Sample A Q5 问 Java 程序的某一行是否 never executed，要证 undecidable。做法就是上面的"行后置"模式：把目标行放在模拟 HALT 之后，让 HALT 的 yes/no 控制目标行是否执行。

卷面写法：构造 N：ignore input; simulate M on w; if simulation halts, execute line n; else forever loop. Prove <M,w>∈HALT iff N reaches line n.

常见失分点：目标 line 放错位置——放在模拟之前就总是执行，和 HALT 无关。iff 证明必须两个方向都写。

标准答案（先写结论）：证书是 k 个学生的集合；verifier 二重循环检查每一对是否有 friend edge。

为什么这是对的：证明 in NP 的本质：yes instance 有一个"短证据"让你在多项式时间内验证。对于 clique，证据就是那 k 个学生。检查：(1) 真有 k 个；(2) 每对学生之间都有边。检查 (2) 需 C(k,2) = O(k²) 次边查找，因 k≤n，整个过程 O(n²)，是输入大小的多项式。

题目语境怎么用：在 Sample B Q6A 里，先证 CARA clique ∈ NP——给证书和多项式 verifier。Q7 再证 NP-hard。

卷面写法：Certificate: a set S of k students. Verifier: check |S|=k; for each pair u,v∈S check (u,v)∈E; accept iff all pass. O(k²) edge lookups, polynomial.

常见失分点：只写"guess a clique"而不描述 verifier 的具体检查步骤，或者忘记论证检查时间是多项式。

内容理解题（先做 3 题）

核心答案（先确认概念）：先假设相反结论成立，例如假设语言 regular 或 decidable，再推出和 pumping lemma 或 HALT 结论冲突。

为什么这是对的：反证法是 CS605 最常用的证明骨架。逻辑链：假设要证明的结论为假（如"L is regular"），利用这个假设导出已知为假的结论（如"a^n b^n 能被 pump"或"HALT is decidable"）。因为已知结论为假，原始假设不可能成立。在 pumping 题里：假设 L regular → 存在 p → 选 witness → 论证任意 pump 破坏语言条件 → 矛盾于 pumping lemma。

题目语境怎么用：遇到 pumping 题，开头写"Assume, for contradiction, that L is regular."遇到 undecidability 题，开头写"Assume there exists a decider D for L."这句假设是后面一切推导的起点。

卷面写法：假设句 → 利用假设推出中间构造 → 分析构造导致矛盾 → 结论"Therefore the assumption is false"。必须明确点出矛盾在哪——不能只说"矛盾"，要说"这 contradicts the pumping lemma which requires xy^iz ∈ L"。

常见失分点：假设写对了但后面忘记回到假设去推矛盾——比如 pumping 完后只说"so the string is not in L"但没说"this contradicts the pumping lemma"。必须明确点出矛盾点。

核心答案（先确认概念）：因为你不能挑自己喜欢的切法；你要证明所有合法切法 pump 后都会破坏语言条件。

为什么这是对的：Pumping lemma 说的是"存在一种切法 xyz"使得 pump 后还在语言里。你要反驳它，就要证明"对所有合法切法"，pump 后都破坏条件。如果只挑了一种切法（比如 y 恰好是最方便的那几个字符），只证明那种切法不行——但 lemma 允许 adversary 选一种行的切法。必须用 |xy|≤p 和 |y|>0 约束确定 y 可能落在哪些区域，然后对每个区域论证 pump 后都坏。

题目语境怎么用：选好 witness word 后，用 pumping 约束推断 y 只能由哪些字符组成。比如选 w=a^p b^p，由于 |xy|≤p，y 只能全由 a 组成。如果 w 更复杂，可能需分情况讨论。

卷面写法："For any split w=xyz with |xy|≤p and |y|>0, by the choice of w, we must have y consisting only of a's. Then consider i=0..." 如需分情况，逐一列出每种区域下 pump 后的结果。

常见失分点：自己挑一个方便 split 而不是说明所有 split 都会坏。也别忘了用 |y|>0——如果 y 是空的，pump 没有变化，矛盾推不出来。

核心答案（先确认概念）：它证明翻译前后的 yes/no 完全同步。没有 iff，decider 不能可靠地从新问题答案推出原问题答案。

为什么这是对的：Reduction 的核心是 f: A → B 满足 x∈A ⇔ f(x)∈B。两个方向：(→) 原问题 yes ⇒ 构造出的实例也 yes；(←) 构造实例 yes ⇒ 原问题也 yes。缺一个方向，归约就断了——decider for B 的 yes/no 不能可靠翻译回 A。比如只证了 M(w) halts → N 有 property，但 N 有 property 可能来自别的原因，那 decider B 说 yes 时不代表 M(w) halts。

题目语境怎么用：在 HALT reduction 题里，构造 N 时让它先模拟 M(w)。(→) M halts ⇒ N 在模拟后触发 property。(←) N 触发 property ⇒ 回顾 N 结构，property 只在模拟 M 停机后出现 ⇒ M halts。两个方向都要在卷面上明确写出。

卷面写法：分开写两个方向：(⇒) Assume <M,w>∈HALT. Then M halts, so during simulation... f(<M,w>)∈L. (⇐) Assume f(<M,w>)∈L. By construction of N... M must halt, so <M,w>∈HALT.

常见失分点：只写一个方向，或两个方向都写得很模糊。方向写反：在 undecidability 归约里是 HALT ≤m L，写成 L ≤m HALT 就反了。

2. FA 和 Regular Languages：有限记忆能做到什么

Finite Automaton 可以理解成“只有有限个状态的读字符串机器”。它从左到右读输入，每读一个字符就换状态。最后停在 accepting state 就接受，否则拒绝。

部件白话解释小例子
State机器当前记住的有限信息。“目前看到偶数个 1”或“目前看到奇数个 1”。
Start state还没读输入时所在位置。通常画一条箭头指进去。
Transition读到某个字符后去哪个状态。在偶数状态读到 1，切到奇数状态。
Accepting state输入读完后若停在这里，就说字符串属于语言。识别偶数个 1 时，偶数状态是 accepting。

Regular language 的直觉

Regular language 是 FA 能识别的语言。它适合固定模式、有限历史、模数信息，比如“以 01 结尾”“包含偶数个 1”“没有连续 00”。它不擅长无限精确计数，比如“a 的数量必须等于后面 b 的数量”。

从会画图到会证明

  1. 先画能识别的简单语言。例如 even number of 1s。你会看到状态就是“机器能记住的摘要”。
  2. 再问机器记不住什么。{a^n b^n : n ≥ 0} 需要记住任意多的 a 数量，FA 状态有限，直觉上不够。
  3. 最后用 pumping lemma 把直觉变成证明。有限状态意味着足够长的字符串读到中途必定重复某个状态，重复的那段就可以 pump。

内容理解题（先做 3 题）

核心答案（先确认概念）：FA 只有有限状态，能记有限摘要和模数信息，但不能准确保存任意大的计数。

为什么这是对的：FA 的"记忆"就是它的当前状态。因为状态数是固定的（比如 5 个或 100 个），它能表达的信息也是有限的。它能区分"读到偶数个 1"和"读到奇数个 1"（只需 2 个状态）；但不能区分"读到 15 个 a"和"读到 16 个 a"——因为要做到后者，需要至少 17 个不同状态逐个区分每种数量，一旦 n 可以任意大，需要的状态数就无限了。这就是为什么 FA 能处理模运算和局部模式，但处理不了需要精确计数的语言。

题目语境怎么用：看到题面问"这个语言是否 regular"，先看 yes 条件是否需要"等量比较"或"任意长精确计数"。如果需要，大概率 not regular——选 pumping lemma。如果只是判断后缀、模数、禁止局部模式——那很可能 regular，画个 FA 就行。

卷面写法：若证 regular：画出 DFA/NFA，说明每个状态含义。若证 not regular：用 pumping lemma，选 witness word 使 pump 后破坏数量相等关系。

常见失分点：把"FA 状态有限"当成"FA 不能处理任何需要计数的东西"——实际上 FA 可以做模计数（如除以 3 的余数），不能做的是任意大精确计数。

核心答案（先确认概念）：有限记忆就能识别的字符串集合，例如固定后缀、奇偶性、有限模式。

为什么这是对的：Regular language 的精髓是"你只需要记住有界的信息"来判断字符串是否属于语言。比如"以 01 结尾"，只需记住最后两个字符——更早的字符不影响判断。比如"包含偶数个 1"，只需记住当前 1 个数的奇偶性——只有两个可能值。更广地说，regular language 等价于能被正则表达式描述的语言，也等价于能被有限状态机识别的语言——三条路通向同一个集合。

题目语境怎么用：如果题目让你"describe this language in English"且语言定义里有 # 分隔符、block 结构等，先判断它是否 regular。直觉说 regular → 试着画 DFA；直觉说 not regular → 准备 pumping。

卷面写法：遇到"判断 regular 与否"的题，结论写"This language is (not) regular because..."后跟 pumping 证明或 DFA 构造。

常见失分点：以为"包含复杂结构"就不是 regular。实际上很多看起来复杂的字符串模式其实只需要有限状态（比如特定分隔符之间的模式匹配），关键仍然是"需要记住的信息量是否有限"。

核心答案（先确认概念）：它需要比较任意多的 a 和 b 数量，有限状态没有足够记忆。

为什么这是对的：要判断 a^n b^n，需要先数清前面有多少个 a，再检查后面 b 的数量是否相等。n 可以任意大——可以是 5、1000、一百万。FA 只有固定数量状态，假设 k 个，当 n > k 时，读 a 的过程中必然回到某个之前访问过的状态（鸽子洞原理）。一旦状态重复，FA 就"忘了"读了多少个 a——它无法区分"读了 15 个 a"和"读了 15+m 个 a"。这就是 pumping lemma 的直觉根源：状态有限 + 输入足够长 ⇒ 必然有循环 ⇒ 循环可以 pump。

题目语境怎么用：a^n b^n 是 CS605 里 not regular 的"hello world"。选 w = a^p b^p，用 |xy|≤p 锁死 y 在 a 区域，pump 后 a 和 b 数量不等。这个模式是所有后续 pumping 题的模板。

卷面写法：Assume L = {a^n b^n} is regular. Let p be pumping length. Choose w = a^p b^p ∈ L. For any split w=xyz with |xy|≤p, |y|>0, y consists only of a's. Then xy^0z = a^{p-|y|} b^p ∉ L. Contradiction.

常见失分点：选 w = a^p b^p 是对的，但忘记用 |xy|≤p 来限制 y 的位置。没有这个限制，y 可能跨过 a 和 b 边界，导致分析变复杂。

为什么这是对的：这道题问"给定一个 FA，它是否接受某个长度大于 5 的字符串"。因为 FA 只有有限个状态（设为 n），任何长度大于 n 的 accepting path 必然包含状态循环。你不需要搜索无限长的路径——只需检查是否存在从 start state 到 accept state 的路径长度 ≥ 6。可以用 BFS/DFS 系统搜索，由于状态有限，搜索必然终止。这三步（验证编码 → 搜索 → 终止论证）构成了 decidable 的完整证明。

题目语境怎么用：Sample A Q2(a) 要求证明这个语言 decidable。步骤：(1) 检查输入是否合法 FA 编码——不是就 reject；(2) 用图搜索找长度 ≥6 的 accepting path；(3) 论证搜索因状态有限而终止。

卷面写法：On input <M>: (1) Validate encoding, reject if invalid. (2) Search FA state graph for an accepting path of length ≥ 6. (3) Since M has finitely many states, search terminates. Accept if such path exists, else reject. Thus L is decidable.

常见失分点：忘记先验证输入合法性。TM 的输入可能是任何字符串，非法编码应 reject。另外需要论证搜索终止——利用"无环路径长度最多等于状态数"或类似分析。

标准答案（先写结论）：从 start state 做 graph reachability，看是否能到达 accepting state；状态有限，所以会停机。

为什么这是对的：判断 FA language nonempty 等价于问"有没有一条从 start state 到 accept state 的路径"。这是标准图可达性问题——从 start state 做 BFS/DFS，标记所有可达状态，检查是否有 accept state 被标记。因为 FA 状态数有限，标记过程必然终止。如果 accept state 可达 → 存在被接受的字符串 → language nonempty；如果都不可达 → language empty。和 Sample A Q2(a) 一样，核心都是"FA 状态有限 ⇒ 搜索终止 ⇒ decidable"。

标准答案（先写结论）：证书给出那个 word；verifier 检查 word 在 W 中且每个 automaton 都接受它。

卷面写法：Certificate: a word w. Verifier: (1) Check w∈W. (2) For each FA Ai, simulate Ai on w. (3) If all accept, accept. O(m·|w|) steps, polynomial. Thus L∈NP.

为什么这是对的：HALT 归约中的"后置触发"技巧。构造 N：ignore input → simulate M on w → if simulation halts, execute line n. 于是 M(w) halts ⇔ line n is executed。若存在 decider D 能判断任意程序某行是否 never executed → 把 N 喂给 D → D 答案直接告诉你 M(w) 是否 halts → 判定 HALT → 矛盾。

3. Regular Pumping Lemma：证明 FA 不够用

Pumping lemma 的中文直觉是：如果一个语言真的 regular，那么每个足够长的 yes 字符串里，都有一小段可以重复或删除，重复后仍然应该是 yes。我们要证明 not regular，就找一个 yes 字符串，让任何合法小段一动就变成 no。

Regular pumping 里的每个条件

条件人话写证明时怎么用
w=xyz把选中的字符串切成三段。你不能指定所有切法，只能利用约束推断 y 在哪里。
|xy| ≤ px 和 y 都在前 p 个字符里。若选 w=a^p b^p，则 y 只能由 a 组成。
|y| > 0被 pump 的段不是空的。删除或重复 y 一定会改变某些字符数量。
xy^iz ∈ L无论 pump 几次都还应在语言里。找 i=0 或 i=2 让它坏掉。

最小 worked example：{a^n b^n : n ≥ 0}

  1. 假设这个语言 regular，于是有 pumping length p。
  2. 选 w=a^p b^p。这个字符串在语言里，长度也足够长。
  3. 任意合法切法 w=xyz 满足 |xy|≤p，所以 y 只能落在前面的 a 区域。
  4. 因为 |y|>0，删掉 y 后，a 变少，b 仍是 p 个。
  5. xy^0z 不再满足 a 和 b 数量相等，矛盾。所以语言 not regular。

考试模板：Assume L is regular. Let p be the pumping length. Choose a carefully designed w∈L with |w|≥p. For any split w=xyz satisfying |xy|≤p and |y|>0, show where y must lie. Pump with i=0 or i=2. The pumped word violates the language definition, contradiction. Therefore L is not regular.

为什么这是对的：Pumping lemma 是 regular language 的必要条件：如果 L 是 regular ⇒ L 满足 pumping property。逆否命题：如果不满足 pumping property ⇒ 不是 regular。所以 pumping lemma 只能"证伪"——证明某个语言不 regular。不能用它证明语言是 regular（因为有些 non-regular 语言也能被 pump——pumping lemma 只是必要条件，不充分）。证 regular 的方法永远是构造 FA 或正则表达式。

题目语境怎么用：Q1 通常"prove L is not regular"。结构：(1) 假设 L regular，(2) pumping lemma 给 p，(3) 选 witness w，(4) 分析所有 split 导致矛盾，(5) 结论 L not regular。

卷面写法：严格按五步：Assume regular → Let p be pumping length → Choose w = ... → For any split... → Pump... → Contradiction → Therefore not regular。

常见失分点：试图用 pumping lemma 证明语言是 regular（方向反了）。或忘记说"assume L is regular"，直接跳到选 w——阅卷人不知道你的假设。

为什么这是对的：这个条件来自 pumping lemma 的证明：p 是 FA 的状态数，第一个重复状态必然在前 p+1 步内出现。所以循环（对应 y）必定在输入的前 p 个字符内。证明中利用这个条件把 y 锁死在狭窄区域：比如 w = a^p b^p，因为 |xy|≤p，xy 最多覆盖到第 p 个字符——前 p 个全是 a——所以 y 只能全由 a 组成。没有这个条件，y 可能横跨 a 和 b，分析会复杂得多。

常见失分点：选了 witness 但没用 |xy|≤p 限制 y 的位置。比如选了 w = a^p b a^p，说"y can be anything"——这没有利用 pumping lemma 给你的武器。

为什么这是对的：Pumping lemma 说 xy^iz ∈ L 对所有 i≥0 都成立。你要找某个 i 让结果不在 L 里。i=0（删除 y）常用于破坏"必须足够多"的条件——如 a^n b^n 中删除若干 a 让 a 比 b 少。i=2（重复 y）常用于破坏"不能太多"的条件——如语言要求"a 数量 ≤ 10"，pump 后超了。选哪个 i 取决于你要让数量变多还是变少。大多数经典题中 i=0 或 i=2 就够。

为什么这是对的：L1A 要求存在某个奇数位置 block i 和下一个 block i+1 长度相等。要证 not regular，选 witness 使：(1) 本身在 L 中（满足某对 block 等长），(2) 无论怎么 pump，这个相等关系都被破坏。好策略：选很多 block，每个长度都不同且都很大——这样 |xy|≤p 把 y 限制在最前面的几个 block 内。如果只让一对 block 等长而其他都不等，pump 改变某 block 长度就会破坏唯一那对相等。

标准答案（先写结论）：说明任意合法 split xyz 中 y 被 pumping length 限制在特定区域。

常见失分点：忘了说"for any split"——只选一种切法然后直接跳到 pump。这是 pumping 证明最严重的失分点，丧失了全称量词。

题目语境怎么用：pumping 证明最后，需要一句话把矛盾说明白。不要说"this is a contradiction"——要具体说"this contradicts the pumping lemma which requires xy^iz ∈ L for all i"。

常见失分点：只说"hence contradiction"不说和什么矛盾。或只说"not in language"忘记指出这与 pumping lemma 要求冲突。矛盾必须两条陈述的冲突，两条都写出来。

卷面写法：Certificate: set S of k phones. Verifier: for each edge (u,v), check u∈S or v∈S. Accept if all edges covered. O(|E|) checks, polynomial. L∈NP.

4. PDA 和 CFL：一个 stack 能多做什么

Pushdown Automaton 是 FA 加一个 stack。Stack 像一摞盘子，只能从顶上放和拿。这个额外记忆让 PDA 可以处理某些配对和嵌套结构，例如 a^n b^n 或括号匹配。

模型能记住什么典型语言边界
FA有限状态。以某后缀结尾、奇偶性。不能准确匹配任意大数量。
PDA有限状态 + 一个 stack。a^n b^n、平衡括号。难以同时比较三段或交叉依赖。
TM可读写的无限纸带。一般算法能处理的问题。仍然有不可判定问题。

CFL pumping 和 regular pumping 的差别

Regular pumping 切成 xyz，只 pump 中间一段 y。CFL pumping 切成 uvxyz，同时 pump v 和 y。原因是 context-free derivation 里可能有两处一起重复。

CFL 条件怎么理解为什么考分情况
w=uvxyz把字符串切成五段。要跟踪 v 和 y 可能落在哪些区间。
|vxy|≤p中间三段总长度不超过 p。它们只能覆盖局部区域，不能横跨太远的多段。
|vy|>0v 和 y 至少有一个非空。pump 后一定有变化。
uv^ixy^iz∈L两段一起重复仍应在语言里。找一个 i 打破数量或顺序关系。

not CFL 的思路

常见 not CFL 语言有三段相等或复制结构，例如 {a^n b^n c^n : n ≥ 0}。选 w=a^p b^p c^p 后，vxy 长度不超过 p，所以它不可能同时覆盖三段。pump 后最多改变一段或两段，三段相等会被破坏。

过关小题：解释为什么 |vxy|≤p 是关键。它不是普通长度限制，而是在强迫 v 和 y 只能影响局部，不能同时“同步修正”所有需要相等的区域。

内容理解题（先做 3 题）

核心答案（先确认概念）：多了一个 stack，可以记一类后进先出的配对信息，例如括号或 a^n b^n。

为什么这是对的：Stack 的本质是"后进先出"——最后放进去的最先被拿出。天然适合嵌套配对：遇到左括号 push，遇到右括号 pop 检查匹配。a^n b^n 同理：读 a 时 push，读 b 时 pop。关键限制：只能访问栈顶，不能跳到栈中间去读。所以 a^n b^n c^n（需要同时比较三个数量，或先比较 a 和 b 再比较 b 和 c）一个 stack 不够。一个 stack = 一种"一维"记忆，处理不了需要二维交叉比较的问题。

题目语境怎么用：分析一个语言是否 CFL，先想能不能用一个 stack 解决。可以（a^n b^n、ww^R、平衡括号）→ 通常是 CFL。需要两个 stack 或随机访问记忆（a^n b^n c^n、ww）→ 通常不是 CFL——用 CFL pumping lemma 证明。

卷面写法：证 CFL：描述 PDA 的 push/pop 策略。证 not CFL：用 CFL pumping lemma，选五段切法 uvxyz，分情况讨论 v 和 y 落在哪里。

常见失分点：认为 PDA 有一个 stack 所以能处理任何需要"记忆"的问题。实际上一个 stack 有严格访问限制（只能看栈顶），很多看似只需记忆的问题需要随机访问。

核心答案（先确认概念）：Context-free 结构可能有两处一起重复，所以写成 uvxyz，同时 pump v 和 y。

为什么这是对的：CFL pumping 来源于 CFG 的推导树。如果字符串足够长，推导树中必然有一个从根到叶的路径上出现两次同一个非终结符 A。这意味着 A ⇒* vAy（产生 v 和 y 两段）且 A ⇒* x（中间一段）。整个推导可循环：uv^i x y^i z。所以 v 和 y 同时被 pump——它们来自同一个递归变量的两次展开。Regular pumping 只有一个 y 是因为 FA 循环在状态图上只产生一段重复。

题目语境怎么用：做 CFL pumping 题时，分析对象是两个泵段 v 和 y。关键是 |vxy|≤p 约束了它们的位置范围——分析 vxy 可能落在 witness 的哪些区域，对每种区域分别论证 pump 后破坏语言条件。

卷面写法：证 not CFL：Assume L is CFL. Let p be CFL pumping length. Choose w = ... ∈ L, |w| ≥ p. For any split w=uvxyz with |vxy|≤p and |vy|>0, consider cases where vxy falls...

常见失分点：用 regular pumping 的 xyz 三切法去证 not CFL——那是错的。CFL 必须用五切法 uvxyz。

核心答案（先确认概念）：因为 v 和 y 可能落在不同区间，你要证明每种合法位置 pump 后都会坏。

为什么这是对的：以 a^n b^n c^n 为例，w = a^p b^p c^p。vxy 长度 ≤p，最多覆盖两个相邻区域（a-b 或 b-c），不能同时覆盖 a、b、c 三个区域。所以分情况：(1) vxy 全在 a → pump 后 a 数量变，三段不等；(2) 全在 b；(3) 全在 c；(4) 跨越 a-b 边界 → a 和 b 可能同时变但 c 不变；(5) 跨越 b-c。每种情况都要论证 pump 后至少一个字符数量与其他不同。这是在行使 for all split 的量词职责。

题目语境怎么用：witness word 含多个不同字母区块时，用 |vxy|≤p 作为"距离限制"分情况。情况划分依据是 vxy 区间覆盖了哪几个字母区域。

卷面写法：列出所有可能的区域覆盖情况（"Case 1: vxy entirely within the a's..."），对每种情况取 i=0 或 i=2，指出 pump 后为什么不在语言里。

常见失分点：漏掉跨越边界的 case——常只分析 vxy 在单个区域。或用 |vxy|≤p 论证跨越情况时分析不彻底。

为什么这是对的：在 CFL pumping 中，|vxy|≤p 是关键的"距离限制"。当你选 w = a^p b^p c^p（长度 3p），vxy 最多只有 p 个字符长，意味着 vxy 无法同时穿越全部三个区域——最多覆盖两个相邻区域的交界处。这避免了 adversary 让 v 和 y 分别增加 a 和 c 而保持与 b 同步的场景。没有这个约束，CFL pumping lemma 在 a^n b^n c^n 这种经典例子上推不出矛盾。

为什么这是对的：当 v 和 y 都在同一个字母区域（比如都在 a 区）时，pump 只改变该区域的字符数量，其他区域不变。如果语言要求多区域等量（如 a^n b^n c^n），只改变一个区域就破坏等式。取 i=2：a 的数量变成 p+|v|+|y|，而 b 和 c 还是 p 个——三段不等。取 i=0：a 的数量变成 p-|v|-|y|，仍然不等。这种"同区域"是最简单的 case，基本上任意 i≠1 都奏效。

5. Turing Machine：从“识别模式”到“一般计算”

Turing Machine 是抽象版计算机。它有一条可以读写的纸带，一个读写头，一组状态，以及根据当前状态和当前符号决定下一步的规则。它能模拟普通程序，因此后面讨论“有没有算法”时，都以 TM 为标准模型。

Decider 和 recogniser 的差别

概念yes 输入no 输入记忆句
Decidableaccept and haltreject and halt永远会给 yes/no 答案。
Turing-recognisableaccept and haltmay reject or loop forever保证认出 yes，不保证处理完 no。
co-Turing-recognisable从 complement 角度看。能认出 no 的一侧。L̄ 是 T-r。

构造 decider 的小台阶

  1. 确认输入是有限对象。例如 finite automaton、word、graph、formula 都是有限编码。
  2. 说明先检查编码合法。非法编码可以直接 reject，这样机器对所有输入都有定义。
  3. 给出会结束的检查过程。FA 状态有限，word 长度有限，图节点有限，所以循环不会无限跑。
  4. 写 accept/reject 条件。过程满足题目 yes 条件就 accept，否则 reject。
  5. 最后补一句 halt on every input。这句是 decidable 题的生命线。

构造 recogniser 的小台阶

Java program eventually prints something、TM eventually accepts something 这类题，常常只能模拟。只要模拟看到目标事件，就 accept；如果事件永远不发生，模拟就永远继续。这就是 Turing-recognisable 的自然形状。

Recogniser 模板：On input encoding of a program or machine, simulate it step by step. If the required event is observed, accept. If the event never occurs, the simulation may run forever. Therefore every yes instance is eventually accepted, so the language is Turing-recognisable.

核心答案（先确认概念）：Decider 对 yes/no 都会停机给答案；recogniser 只保证 yes 时 accept，no 时可以永远跑。

为什么这是对的：这个区别是 CS605 可计算性核心。Decider（判定器）是始终停机的 TM——任何输入最终输出 accept 或 reject。Recogniser 弱一些：yes instance 保证被接受，但 no instance 可能被 reject 也可能永远跑。经典例子：HALT 是 Turing-recognisable（模拟 M(w)，停机就 accept），但不是 decidable（没有算法对所有 <M,w> 都停机正确判断）。口诀："decider = always halts；recogniser = halts on yes"。若一个语言既是 T-r 又是 co-T-r，它就是 decidable——并行跑两个 recogniser，总有一个会停机。

6. Undecidability 和 Mapping Reduction：把 HALT 翻译过去

HALT 是核心已知难题：给你一台机器 M 和输入 w，问 M 在 w 上会不会停机。结论是没有一个 decider 能对所有 <M,w> 都正确回答。

Mapping reduction 的意思是：把一个已知难题的每个输入，机械地改造成新问题的输入。如果新问题能被判定，旧难题也能被判定。旧难题不可能被判定，所以新问题也不可能。

Reduction 不是背方向，是看谁当“难度来源”

要证明 L undecidable，通常写 HALT ≤m L。

这读作：HALT 可以被 many-one reduce 到 L。如果我有 L 的 decider，我就能先把 HALT 输入翻译成 L 输入，再调用 L 的 decider，从而解决 HALT。

归约证明的五个台阶

  1. 假设新语言 L decidable。这是假设对方给了你一个万能判断器 D_L。
  2. 拿到 HALT 输入 <M,w>。你的目标是判断 M 会不会在 w 上停机。
  3. 构造新实例。通常构造一个机器或程序 N，让 N 的某个性质正好取决于 M 是否 halt on w。
  4. 证明 iff。<M,w>∈HALT 当且仅当构造出的实例在 L 里。
  5. 推出矛盾。如果 D_L 存在，就能 decide HALT，但 HALT undecidable。所以 L undecidable。

构造机器 N 时怎么想

很多题会要求某个程序有 property，例如“prints hello”、“accepts some input”、“never writes to file”。你可以让 N 先模拟 M(w)。如果模拟停机，就触发题目 property；如果模拟不停机，property 就永远不会出现。这样新问题的 yes/no 被 HALT 控制。

内容理解题（先做 3 题）

核心答案（先确认概念）：常写 HALT ≤m L：如果新语言可判定，就能判定 HALT，从而矛盾。

为什么这是对的：归约的方向是"从已知难题到目标语言"。已知 HALT 是 undecidable 的。如果 HALT ≤m L（HALT 可 many-one reduce 到 L），那么如果存在 L 的 decider，先做归约翻译再调用 L 的 decider，就能 decide HALT——矛盾。所以关键在于方向：已知难题 → 目标语言。反方向 L ≤m HALT 只能说明 L 不比 HALT 难，不能推出 L undecidable（因为 HALT 的 decider 可能拒绝翻译后的实例，但原实例可能不是 HALT 实例）。

题目语境怎么用：undecidability 证明的"五步台阶"：假设 L decidable → 拿 HALT 输入 <M,w> → 构造新实例 N → 证 iff → 推出 HALT decidable 矛盾。

卷面写法：Assume L decidable by D. Construct f(<M,w>) = <N> such that <M,w>∈HALT iff <N>∈L. Then HALT would be decidable (run D on f(<M,w>)), contradiction.

常见失分点：方向写反——证明 L ≤m HALT 不能推出 L undecidable，只能推出 L 不比 HALT 更难。

核心答案（先确认概念）：N 先模拟 M(w)；若模拟停机，就触发题目 property；若不停机，property 不出现。

为什么这是对的：这个套路叫"后置触发"——把目标 property 放在 HALT 模拟之后。N 的结构：(1) 忽略自己的实际输入（如果有的话），(2) 开始模拟 M 在 w 上的运行，(3) 如果模拟结束（M 停机），就执行触发 property 的操作（抛 exception、print、accept、执行某行等），(4) 如果模拟永远不结束，property 永不出现。这样：M(w) halts ⇔ N 触发 property。整个 property 的 yes/no 被 HALT 的 yes/no 精准控制——就像一个开关。

题目语境怎么用：构造 N 时通常先写"ignore its own input"，然后"simulate M on w"，最后"if simulation halts, do X"。"do X"就是你要对齐的 property。

卷面写法：Construct N: on any input, simulate M on w. If simulation halts, [trigger property]. Otherwise run forever.

常见失分点：Property 触发逻辑放在模拟之前——导致 property 总是触发，和 HALT 无关。或忘记"ignore its own input"，N 自己的输入干扰了逻辑。

核心答案（先确认概念）：必须写清楚输入如何翻译、构造是否可计算，以及 yes/no 的 iff 正确性。

为什么这是对的：Mapping reduction 的完整要求：(1) f 必须是 computable（可有效计算）——你不能说"f 翻译 <M,w> 到 N"但不描述 N 怎么构造；(2) iff 必须两个方向都证——M halts ⇒ f(<M,w>)∈L 且 f(<M,w>)∈L ⇒ M halts；(3) 构造必须从任意 <M,w> 出发，不能依赖于知道 M 是否 halts。如果只说"reduce to HALT"而不给构造和 iff，等于没证明——阅卷人不知道你的归约是否存在、是否正确。

题目语境怎么用：任何 undecidability 证明，如果只写了"this reduces to HALT"而没有具体构造和 iff 论证，得分极低。具体构造 + iff 双方向 = 核心分。

卷面写法：(1) Describe the computable mapping f explicitly. (2) Prove ⇒ direction. (3) Prove ⇐ direction. (4) Conclude with contradiction.

常见失分点：只写"reduce to HALT"。必须给出 mapping、computable 理由、iff 两个方向。三个要素缺一不可。

标准答案（先写结论）：让 N 先模拟 M(w)；若模拟停机才执行指定 line，从而 line 是否执行反映 HALT。

为什么这是对的：L = {<P,n> : line n is never executed in P}。构造 f(<M,w>) = <N, n_target>：N 忽略自己的输入 → 模拟 M 在 w 上的运行 → 如果模拟停机，执行第 n_target 行；否则模拟永远跑，该行永远不执行。于是 M(w) halts ⇔ line n_target is executed ⇔ <N, n_target>∉L。若存在 L 的 decider，套在 N 上就能判定 HALT——矛盾。

题目语境怎么用：Sample A Q5 考"某行是否执行"，用"行后置"模式把目标行放在模拟 HALT 之后。

卷面写法：Assume decider D for L. Construct f(<M,w>) = <N, n>: N ignores input; simulates M on w; if halts, execute line n. Then <M,w>∈HALT iff <N,n>∉L. Run D on <N,n> would decide HALT. Contradiction.

常见失分点：目标 line 放错位置（放在模拟之前），失去了和 HALT 的关联。iff 证明必须双向。

标准答案（先写结论）：构造程序让该不变量在 M(w) 是否停机时被保持或被破坏，从而和 HALT yes/no 对齐。

为什么这是对的：本题要求证一个有关"throughout execution"不变量的性质不可判定。"throughout"意味着程序运行的每一步都必须满足 b < mean(c)。构造 N 的思路：让 N 在模拟 M(w) 期间保持 b < mean(c)（通过设置合适的初始值）；一旦模拟停机，N 故意改变 b 或 c 的值破坏这个不变量。于是 M(w) 不 halt ⇒ 模拟永远跑且不变量始终成立 ⇒ <N> 满足 property；M(w) halt ⇒ 模拟结束后不变量被破坏 ⇒ <N> 不满足 property。

题目语境怎么用：Sample B Q5 — "throughout" 类 property 的归约关键：在模拟停机前保持 property，停机后破坏它。

卷面写法：Construct N: ignore input; initialize b,c so b < mean(c)；simulate M on w; if simulation halts, modify b or c to break b < mean(c). Then M halts on w iff N violates the throughout property.

常见失分点：忘记"throughout"要求每一步都成立——需要在模拟前就建立不变量，且在模拟期间不被破坏。仅在结束时检查不够。

标准答案（先写结论）：把目标 line 放在模拟 M(w) 之后；模拟停机才执行该 line，不停机则永远不到。

为什么这是对的："后置触发"模式。N：ignore input → simulate M on w → if halts, execute target line n。M halts ⇔ line n executed。用 decider D 判断 line n 是否 never executed → 若 D says "never"，则 M 不 halt → 若 D says "executed"，则 M halts → 这样就能 decide HALT——矛盾。

题目语境怎么用：Sample A Q5。和前面 Q5 题同样的模板。核心：目标行的位置决定映射关系的方向。

卷面写法：f(<M,w>) = <N, n>: N ignores input; simulates M on w; if halts, execute line n. Then <M,w>∈HALT ⇔ <N,n>∉L. If L decidable, HALT decidable. Contradiction.

常见失分点：构造和 iff 双向必须完整。缺一不可。

标准答案（先写结论）：证书是 k 个学生的集合；verifier 二重循环检查每一对是否有 friend edge。

为什么这是对的：Clique ∈ NP 是基础。证书 = k 学生集合。验证：大小 = k + 每对有边。O(k²) = O(n²) 多项式。

常见失分点：缺少多项式论证。

7. Complexity：从“能不能算”到“算得快不快”

Computability 问有没有算法。Complexity 问算法随着输入变大，要花多少时间。一个问题可能 decidable，但需要指数时间，实际规模稍大就跑不动。

增长速度怎么读

表达人话考试意义
O(n)输入变两倍，时间大致也变两倍。线性扫描。
O(n^2)常见双重循环。仍是 polynomial。
O(n^k)固定幂次的多项式时间。P 和 NP verifier 关注这个。
O(2^n)每增加一个元素，可能翻倍。通常来自枚举所有子集，常被视为不可扩展。

P 和 NP 不要混

P 是 polynomial time solvable：机器自己能在多项式时间内找到答案。NP 是 polynomial time verifiable：如果别人给你一个候选答案，你能在多项式时间内检查它是否正确。

词白话解释例子
Certificate别人递给你的候选答案。clique 题里，certificate 是一组选中的 vertices。
Verifier检查 certificate 是否有效的算法。检查点数够不够，再检查每对点之间有没有边。
Input size输入编码的长度。图题通常看 vertices 和 edges 的数量。
Polynomial time运行时间被某个固定幂次限制。遍历所有边或所有点对通常可接受；遍历所有子集通常不可接受。

证明 in NP 的四个台阶

  1. 说 certificate 是什么。不要只写“guess a solution”。要具体：一组 vertices、一组 numbers、一个 assignment。
  2. 检查格式和大小。例如 certificate 里的点是否都在图中，数量是否至少 k。
  3. 检查题目条件。例如 clique 要检查每对选中点之间都有边。
  4. 说明时间是 polynomial。例如最多检查 k^2 对点，每次查边，整体多项式。

in NP 模板：We construct a polynomial-time verifier V. The certificate is the proposed solution. V checks that the certificate has the right form and size, then checks the condition in the language definition. These checks require only polynomially many steps in the input size, so L∈NP.

内容理解题（先做 3 题）

核心答案（先确认概念）：Computability 问有没有算法；complexity 问算法随着输入变大要花多久。

为什么这是对的：Computability（前半学期）关心"存在不存在一个总能停机的算法解决这个问题"。答案只有 yes（decidable）或 no（undecidable），不关心效率。Complexity（后半学期）在 decidable 的前提下进一步问："这个算法需要多长时间？"——多项式时间（efficient）、指数时间（infeasible）等。一个问题可能 decidable 但需要指数时间，实际规模稍大就跑不动。P vs NP 的核心就是"能快速验证的问题是否也能快速求解"。

题目语境怎么用：看到题面有"polynomial time""certificate""verifier""NP"等词，就进入 complexity 领域。看到"decidable""undecidable""recognisable""HALT"就进入 computability 领域。不要在两个领域间混用工具——比如用 HALT reduction 证明 NP-hardness 是错的。

卷面写法：Complexity 题：给出 verifier，分析 runtime，证多项式。Computability 题：给 decider/recogniser，或从 HALT 归约。

常见失分点：在 complexity 题里写"reduce to HALT"——这是 computability 工具，不能证 NP-hard。NP-hard 归约是从已知 NP-complete 问题做多项式时间归约。

核心答案（先确认概念）：P 是能快速求解；NP 是给你候选答案后能快速验证。

为什么这是对的：P：存在一个多项式时间算法，对任何输入自己找到答案。NP：如果别人告诉你一个候选答案（certificate），你可以在多项式时间内验证它是否正确。关键不对称：NP 不要求"找到"答案快，只要求"检查"快。比如 clique：找到 k-clique 可能很慢（试遍所有 C(n,k) 组合），但验证"给定 k 个点是不是 clique"很快（检查每对有没有边）。P ⊆ NP（能快速求解的必然能快速验证——直接自己解一遍然后比较），但 NP ⊆ P 是著名的 open problem。

题目语境怎么用：证明 in NP 永远用 certificate-verifier 模型：先定义 certificate 是什么（一组点、一个 assignment、一个子集等），再描述 verifier 如何多项式时间检查。不要写"guess a solution"——那是 NTM 描述，阅卷人期望 certificate-verifier。

卷面写法：To prove L ∈ NP: give a polynomial-time verifier V. On input (x, c) where c is the certificate: [verification steps]. V accepts iff x ∈ L. V runs in polynomial time.

常见失分点：把"guess a solution and verify in polynomial time"当作 NP 证明——应该给具体的 certificate 和具体的 verifier 检查步骤。还常见的问题是混淆 P 和 NP 的定义方向。

核心答案（先确认概念）：通常是一组选中的点、边、集合或赋值，用来作为 yes instance 的候选证据。

为什么这是对的：Certificate 是和具体问题绑定的"短证据"。clique 的 certificate = 一组 k 个顶点。vertex cover = 一组 k 个顶点。Hamiltonian path = 一个顶点排列。3-SAT = 一个 true/false assignment。集合划分 = 一个子集。规律：certificate 总是 yes instance 的特征结构——如果答案 yes，就存在这样一个结构。Verifier 的作用就是按语言定义逐条检查这个结构是否合格。

题目语境怎么用：Sample 题中证书例子：Sample A Q6A 两个 subsets 覆盖 A 且 sums equal → certificate = 两个子集 Y,Z；Sample A Q6B 一组 FA 都接受某 word → certificate = 那个 word；Sample B Q6A clique → certificate = k 个学生；Q6B vertex cover → certificate = k 个 phone。

卷面写法：明确写："The certificate is a [specific structure]. The verifier checks [specific conditions in language definition]." 要具体，不要说"the solution"——说清楚它是什么数据结构。

常见失分点：Certificate 定义太模糊——"the solution""the correct answer"。阅卷人需要知道具体是什么：多少个点？什么格式？有没有大小限制？

标准答案（先写结论）：一个 size k 的学生子集 S。Verifier 检查每对学生是否在 E 中。

为什么这是对的：Certificate = 声称构成 clique 的 k 个学生集合。Verifier 三步：(1) 检查 |S| = k；(2) 检查所有学生都在 V 中；(3) 对每对 u,v∈S 检查 (u,v)∈E。如果都通过，accept。C(k,2) = O(k²) 次边查找，k ≤ |V| = n，所以 O(n²)，多项式时间。这三步是 clique ∈ NP 的标准证明。

题目语境怎么用：Sample B Q6A 是 Q6 的第一部分——证明 clique ∈ NP。后面 Q7 用 3-SAT 归约证 NP-hard，从而完成 NP-complete 证明。

卷面写法：Certificate: a subset S of k students. Verifier: (1) |S|=k; (2) for each u≠v in S, verify (u,v)∈E; (3) accept iff all checks pass. Runtime O(k²) ⊆ O(n²). Therefore L6A ∈ NP.

常见失分点：遗漏 |S|=k 检查；或不分析 runtime 多项式。三项（证书、验证步骤、runtime）缺一不可。

标准答案（先写结论）：一个 size k 的 phone 子集 S。Verifier 检查每条 communication edge 至少有一个端点在 S 中。

为什么这是对的：Vertex cover 的 certificate = 声称覆盖所有边的 k 个 vertex（phone）。Verifier：(1) 检查 |S| = k；(2) 遍历每条边 (u,v)，检查 u∈S 或 v∈S；(3) 如果每条边都被覆盖，accept。遍历 |E| 条边，每次检查集合隶属 O(1)（用哈希集），总 O(|E|) = O(|V|²) 最坏，多项式。

题目语境怎么用：Sample B Q6B 和 Q6A 是同时的——一个证 clique ∈ NP，一个证 vertex cover ∈ NP。对比理解：证书都是顶点子集，但验证条件不同。

卷面写法：Certificate: a set S of k phones. Verifier: (1) |S|=k; (2) for each edge (u,v), check u∈S or v∈S; (3) accept iff all edges covered. O(|E|) polynomial. L6B ∈ NP.

常见失分点：写成"每条边两个端点都在 S 中"——那是 vertex cover 的条件反了，实际是"至少一个"。或写成 dominating set（覆盖所有 vertex，而非所有 edge）。

标准答案（先写结论）：Certificate 是两个子集 Y,Z；verifier 检查 proper、覆盖每个元素、并比较两边 sum。

为什么这是对的：这道题要求把集合 A 分成两个 proper subset Y 和 Z（Y∪Z=A，且两个都 proper），且 sum(Y)=sum(Z)。Certificate = 两个子集 Y 和 Z。Verifier 检查：(1) Y⊂A 且 Z⊂A（proper——排除 Y=A 或 Z=A）；(2) Y∪Z = A（覆盖所有元素——每个 A 的元素至少在一个子集中）；(3) sum(Y) = sum(Z)（两边元素值求和相等）。每步都是 O(|A|) 或 O(|A|log|A|)，多项式时间。

题目语境怎么用：Sample A Q6A — 集合划分相关题。Certificate 不止一个集合（不像 clique），而是两个集合 Y 和 Z。注意 proper 条件——排除平凡解（Y=A, Z=∅ 之类的）。

卷面写法：Certificate: two subsets Y,Z ⊆ A. Verifier: (1) Y ⊂ A, Z ⊂ A (proper); (2) Y∪Z = A; (3) sum(Y) = sum(Z). All checks O(|A|). Polynomial. L ∈ NP.

常见失分点：漏掉 proper subset 检查——如果 Y=A, Z=∅，Y∪Z=A 仍然满足，但这不合 proper 要求。还有覆盖检查（每个元素至少在一个子集中）不能漏。

标准答案（先写结论）：模拟 J 每一步，读变量 a,b,c；一旦 a+b=c 成立就 accept。

为什么这是对的：逐行模拟 J 的执行。每步后检查 a+b=c。满足 → accept 停机。J 终止但不满足 → reject 停机。J 无限循环且不满足 → TM 也无限循环。yes instance：条件最终发生，模拟最终到达那一刻，accept。no instance：可能 reject（J 终止）或 forever loop（J 无限循环）。这是标准的 Turing-recognisable 构造。

题目语境怎么用：Sample A Q2(b) — recognisable but not decidable。对比 (a) FA 题 decidable，深入理解两者区别。

标准答案（先写结论）：构造程序先模拟 M(w)；根据是否 halts 安排是否抛 exception 21，让 property 与 HALT 对齐。

8. NP-complete：既在 NP，又至少和已知难题一样难

NP-complete 证明有两半。第一半是 membership：证明这个问题在 NP。第二半是 hardness：把一个已知 NP-complete 问题，在多项式时间内翻译成它。

部分要证明什么常见扣分点
in NP给 certificate 后能 polynomial-time verify。忘记写 certificate，或忘记说明 runtime。
NP-hard已知 NP-complete 问题可多项式时间 reduce 到本题。方向写反，或者只说“similar to 3-SAT”没有 construction。
Correctness原实例 yes 当且仅当新实例 yes。只证明一个方向。
Polynomial construction翻译过程本身不能指数爆炸。没有说明新实例大小。

用 3-SAT 归约时怎么读

3-SAT 输入是一组 clauses，每个 clause 有 3 个 literals。问题问是否存在一组 true/false assignment，让所有 clauses 都为 true。很多 reduction 会把 variable、literal、clause 变成图里的节点、集合里的元素或选择结构。

  1. 源问题先定清楚。例如从 3-SAT、CLIQUE、VERTEX-COVER 或 HITTING-SET 来。
  2. 说明每个源问题部件变成什么。variable 变成选择 gadget，clause 变成必须满足的约束。
  3. 证明 yes 到 yes。如果原公式可满足，怎样从 satisfying assignment 构造新问题的解。
  4. 证明 yes 反推 yes。如果新问题有解，怎样读回原公式的 satisfying assignment。
  5. 说明规模。每个 variable 和 clause 只产生多项式数量的新对象，所以 reduction polynomial。

NP-complete 模板：First, prove L∈NP by giving a polynomial-time verifier. Second, prove NP-hardness by reducing from a known NP-complete problem A. Given an instance x of A, construct an instance f(x) of L in polynomial time. Show x∈A iff f(x)∈L. Therefore L is NP-hard. Since L∈NP, L is NP-complete.

内容理解题（先做 3 题）

核心答案（先确认概念）：先证明 in NP，再证明 NP-hard。缺一半都不完整。

为什么这是对的：NP-complete 的定义就是"in NP + NP-hard"。两者缺一不可。第一部分（in NP）：给证书和多项式 verifier，证明 yes instance 可以被多项式时间验证。第二部分（NP-hard）：把一个已知的 NP-complete 问题（如 3-SAT、CLIQUE）在多项式时间内归约到你的语言。只有两半都证了，才能说你的语言是 NP-complete。Sample B 的设计正是这样——Q6 证 in NP，Q7 证 NP-hard。

题目语境怎么用：Q6+Q7 是经典的 NP-complete 组合。Q6 做 in NP（给出 certificate 和 verifier），Q7 从已知 NP-complete 问题（通常题目给定，如"3-SAT is NP-complete"）做多项式时间归约。

卷面写法：Part 1 (in NP): certificate = ...; verifier checks ... in O(...) time. Part 2 (NP-hard): reduce from known NP-complete problem A; construct f(x); prove x∈A iff f(x)∈L; f computable in polynomial time.

常见失分点：只做一半——只证 in NP 或只证 NP-hard 都不算 NP-complete。或把 NP-hard 的归约写成 HALT reduction（混用了 computability 和 complexity 的工具）。

核心答案（先确认概念）：从已知 NP-complete 问题 reduce 到要证明的新问题，例如 3-SAT ≤p L。

为什么这是对的：和 undecidability 归约一样，方向是"已知难题 → 目标语言"。已知 3-SAT 是 NP-complete（即 NP-hard）。如果 3-SAT ≤p L（3-SAT 可多项式时间归约到 L），那任何 NP 问题都可以先归约到 3-SAT 再归约到 L——所以 L 也是 NP-hard。反方向（L ≤p 3-SAT）只能说明 L ∈ NP（如果再加 in NP 证明的话）但不能证明 NP-hard。

题目语境怎么用：Sample B Q7 给定"3-SAT is NP-complete"，要求你从 3-SAT 归约到 CARA clique。方向：3-SAT → CARA clique。将 3-SAT 的 formula φ 映射为图 G 和数字 k。

卷面写法：Reduce from 3-SAT: given a 3-CNF formula φ with m clauses, construct a graph G = (V,E) and integer k = m such that φ is satisfiable iff G has a k-clique.

常见失分点：方向写反——把 L 归约到 3-SAT 只说明 L ∈ NP，不是 NP-hard。另外，构造必须在多项式时间内完成（节点数、边数都是多项式）。

核心答案（先确认概念）：原实例 yes 推出新实例 yes；新实例 yes 也能反推出原实例 yes。

为什么这是对的：Reduction correctness 的 iff：(→) φ satisfiable ⇒ G has k-clique。做法：从 satisfying assignment 中每 clause 选一个 true literal，对应 vertex 构成 clique。(←) G has k-clique ⇒ φ satisfiable。做法：k-clique 必须每 clause 恰好一个 vertex（因为 clause 内无边），这些 vertex 对应的 literal 互不冲突（冲突则无边），全设为 true 就是 satisfying assignment。两个方向都证了，归约才完整。缺一个方向，decider for L 的答案不能可靠地翻译回原问题。

题目语境怎么用：Sample B Q7(c) 要求证明 reduction 的 correctness。分两个段落：Paragraph 1 = forward direction；Paragraph 2 = reverse direction。每段都要给出明确的推理链。

卷面写法：(⇒) If φ satisfiable with assignment τ: for each clause, pick one literal true under τ; corresponding vertices form k-clique because true literals never conflict. (⇐) If G has k-clique: since vertices in same clause are not connected, clique must contain exactly one per clause; set those literals true; they don't conflict, so τ is satisfying.

常见失分点：← 方向忘记论证"每 clause 恰好一个 vertex"——需要用到"clause 内部无边"+"clique 大小 = clause 数"。"互不冲突"也需要用"冲突 literal 之间无边"来论证。

考试题目题（再做 5 题）

标准答案（先写结论）：题目给定 3-SAT is NP-complete，因此 reduction 从 3-SAT 到 L6A/CARA clique。

为什么这是对的：NP-hard 归约必须从一个已知 NP-complete 问题出发。题目明确给了"3-SAT is NP-complete"作为已知条件。所以方向是 3-SAT ≤p CARA clique。将任意 3-CNF formula φ 映射为 (G, k)，使得 φ satisfiable iff G has k-clique。如果做成了，因为 3-SAT 是 NP-hard，CARA clique 也是 NP-hard。加上 Q6A 已证 ∈ NP，就完成了 NP-complete。

题目语境怎么用：Sample B Q7(a) 描述 reduction 构造，(b) 给具体公式输出图，(c) 证 correctness 和 NP-complete。

卷面写法：Since 3-SAT is NP-complete, reduce from 3-SAT: given 3-CNF φ with m clauses, construct G with 3m vertices (one per literal occurrence) and k = m.

常见失分点：不知道或忘记引用题目给的已知条件"3-SAT is NP-complete"。或选了错误的源问题。

标准答案（先写结论）：考你是否真的会把 clauses/literals 转成 clique construction 的 vertices 和 edges，而不是只背模板。

为什么这是对的：这道题不再让你描述 reduction 的一般方法，而是给一个具体的 3-CNF formula，要求你输出对应的图（列出 vertices 和 edges）以及 k 值。这是验证你"真的会构造"而不是"只会说构造方法"。你需要：每个 clause 的 3 个 literal → 3 个 vertex（标记为 literal 名）；同一 clause 内的 vertex 之间不加边；不同 clause 的 vertex 之间，如果对应的 literal 不冲突（不是 x 和 ¬x 的关系），就连边。k = clause 数。

题目语境怎么用：Sample B Q7(b) — 给了具体公式如 (b∨a∨c)∧(¬b∨a∨¬c)...，你要画出 vertices 列表和 edges 列表。

卷面写法：Vertices: (b,1), (a,1), (c,1), (¬b,2), (a,2), (¬c,2), ... where (ℓ,i) means literal ℓ from clause i. Edges: (ℓ,i)–(ℓ´,j) for i≠j and ℓ ≠ ¬ℓ´. k = number of clauses.

常见失分点：忘记同一 clause 内部的 literal 之间不加边。或者加边规则搞反——把冲突 literal 连上了。

标准答案（先写结论）：因为 NP-complete 需要 membership + hardness；前一题已经证明 L∈NP，Q7 重点补 NP-hard reduction。

为什么这是对的：NP-complete = in NP + NP-hard。Sample B Q6A 已经证明了 CARA clique ∈ NP（给了 certificate 和 verifier）。Q7 的任务是补上 NP-hard 证明。因此 Q7 的开头应该引用 Q6A 的结论："By Q6A, we already have L6A ∈ NP. It remains to prove NP-hardness." 如果不引用，阅卷人可能认为你忘了 in NP 这一半，或者要求你重新写一遍。

题目语境怎么用：Sample B Q7 — 开头先声明"From Q6, we have L ∈ NP. Now we prove NP-hardness by reduction from 3-SAT."

卷面写法：By Question 6A, L6A ∈ NP. To show NP-complete, we prove L6A is NP-hard. Reduce from 3-SAT...

常见失分点：完全不提 Q6A 的结论，重新写一遍 in NP 证明（浪费时间），或者忘记 NP-complete 需要两半。

标准答案（先写结论）：选足够长的 unary block 串，让 pumping 后某个奇数位置相邻 blocks 不再相等，从而破坏 yes 条件。

为什么这是对的：L1A 的 yes 条件是"存在某个奇数 i 使 wi=wi+1"。选 witness 时策略：(1) 让 block 数 ≥ 2p（保证 |xy|≤p 只能覆盖最早几个 block），(2) 每个 block 长度不同且足够大（如用 p 的倍数偏移），(3) 只让一对 block（比如 w1 和 w2）等长，其他都不等。pump 后改变 w1 或 w2 的长度，等长关系破坏。注意避免 pump 后意外产生新等长关系。

题目语境怎么用：Sample B Q1(a) pumping 题。witness 选择是 pumping 证明中最需要技巧的一步——要"脆弱"到 pump 必坏。

卷面写法：Choose w = a^p # a^p # a^{p+1} # a^{p+2} # ... # a^{p+2p-2} (total 2p blocks). Only w1 = w2 = a^p. |xy| ≤ p ⇒ y entirely within first block(s). Pumping changes w1 or w2, breaking the only equal pair.

常见失分点：witness 让多对 block 等长——pump 后可能意外"修复"，使 pumped word 仍在 L 中。要让语言条件极其脆弱。

标准答案（先写结论）：FA 状态有限，只需系统搜索长度超过 5 的可达 accepting path；搜索空间有限，最后 halt。

为什么这是对的：FA 有 n 个状态。目标：是否存在长度 ≥ 6 的 accepting path。算法：搜索状态图，看 accept state 是否可通过长度 ≥ 6 的路径从 start state 到达。因为有状态循环，长度可以无限——但只要发现一条包含循环且通往 accept state 的路径，就是 yes。具体做法：先查是否有简单路径（无环）长度 ≥ 6 到达 accept；如果有含环路径，任意长都可达，yes。搜索在检查完所有 (state, length_in_mod) 或所有无环路径后必然终止。所以 decidable。

题目语境怎么用：Sample A Q2(a) — FA 相关问题都是 decidable，因为"输入对象本身提供了终止边界"。对比 (b) Java 题。

卷面写法：Search FA state graph for an accepting path of length ≥ 6. Since FA has finitely many states, either (a) a simple path of length ≥ 6 reaches accept → yes, or (b) a cycle exists on the way to accept → infinite paths exist → yes, or (c) no path of length ≥ 6 reaches accept → no. Algorithm halts. L is decidable.

常见失分点：忘记论证搜索终止——只说"搜索有限"没说明为什么。要提到 FA 状态数有限决定了无环路径数量的上界。
