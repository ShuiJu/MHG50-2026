CS603 形式化验证：用逻辑证明程序和系统满足性质

CS603 的核心是"不要只测试几个例子，而是用数学方式证明"。题目分成几块：逻辑表达、Hoare logic、Dafny 程序验证、模型检测和 SAT/SMT 求解。页面先解释符号，再讲每类题怎么下手。

1. 形式化验证在做什么

普通测试问："这几个输入能不能跑对？" 形式化验证问："所有满足条件的输入都能不能保证正确？" 它用逻辑公式写清楚程序前后应该满足什么，再让人或工具证明这些公式。

词 从零理解 例子
Specification程序应该满足的说明。排序后数组递增。
Precondition程序开始前必须是真的条件。数组长度大于 0。
Postcondition程序结束后必须是真的条件。返回值是数组最小值。
Invariant在某个过程反复执行时一直保持真的条件。循环每次开始时 0 <= i <= a.Length。
Verifier检查证明条件是否成立的工具。Dafny verifier、Z3、model checker。
Ghost只用于证明、不进入最终运行程序的变量/代码。Dafny ghost field 记录抽象状态。

过关问答：先内容理解，再 past/sample paper 题，悬停或点击显示答案

内容理解题（先做 3 题）

核心答案（先确认概念）：把正确性要求写成数学性质，并尝试证明所有相关行为都满足，而不只测试几个例子。

为什么这是对的：测试只能覆盖有限输入——测1000个数组也不等于"对所有数组正确"。形式化验证用逻辑公式描述"所有输入"，再用数学推理覆盖无穷情况。比如证明排序正确，你不写 a[k]≤a[k+1] 只测几个例子，而写 ∀k·0≤k<len-1 ⇒ a[k]≤a[k+1] 让verifier穷尽。这就是CS603从逻辑教起的根源：把"程序正确"翻译成可证明的数学命题。

题目语境怎么用：遇到任何程序验证题，先问"对所有合法输入应该保证什么"，然后写precondition和postcondition，不要跳到"跑几个例子没报错"。

卷面写法：答题时先声明specification是什么，再写precondition（输入假设）和postcondition（输出承诺），最后给出invariant或proof obligation作为证明过程。

常见失分点：只描述测试结果（"跑过了"）而不给数学理由；把postcondition当invariant写——invariant是中间状态的性质，postcondition是最终要求，两者混淆导致证明链断裂。

核心答案（先确认概念）：Safety 是坏事不发生；liveness 是好事最终发生。

为什么这是对的：Safety和liveness是互补的时间性质。以交通灯为例：两方向不同时为绿是safety（坏事永不发生，写 G¬(green_NS∧green_EW)），每个红灯最终变绿是liveness（好事最终发生，写 G(red_NS→F green_NS)）。若只写safety不提liveness，系统可能永远不动但"不错"——叫trivial satisfaction。两者必须同时验证系统才有意义。

题目语境怎么用：看题面关键词——"never"、"must not"→safety用G算子；"eventually"、"finally"→liveness用F算子。

卷面写法：写LTL公式先标类别：safety用 G(condition) 或 G¬(bad)；liveness用 G(trigger→F outcome)。每个temporal operator后加自然语言解释。

常见失分点：把liveness写成safety公式，如 G(request→response) 表达"请求后有响应"是错的——它只要求两者同时成立；G(request→F response)才表达"最终会有"。另外把fairness假设当liveness属性写也会丢分。

核心答案（先确认概念）：因为 contract、Hoare、Dafny、model checking、SMT 都需要把要求写成公式。

为什么这是对的：整门CS603的所有技术本质上都在操作逻辑公式。Hoare triple {P}S{Q}本身就是implication。Dafny的requires/ensures是predicate logic嵌入代码。LTL/CTL是加了时间算子的命题逻辑。SAT/SMT求逻辑公式的满足性。如果写不出"数组最小值"的公式 ∀j·0≤j<len⇒min≤a[j]，任何工具都用不了——你不知道该证明什么。

题目语境怎么用：拿到题第一反应不是写代码或注解，而是先用自然语言描述"程序应该做什么"，再翻译成带量词和predicate的公式。

卷面写法：先定义每个predicate的含义和参数类型（如Student(x)中x的类型域），再写完整公式，标注每个量词的取值范围。

常见失分点：跳过逻辑表达直接写Hoare/Dafny注解，导致invariant和postcondition间缺少逻辑连接；量词范围未声明让公式含义模糊。

考试题目题（再做 5 题）

标准答案（先写结论）：如果程序终止，则返回 min 满足 postcondition，例如不大于数组每个元素。

为什么这是对的：Partial correctness等于"假设终止则结果对"。Find循环初始化i=1,min=a[0]时invariant"min是a[0..i)最小值"自然成立（单元素集合）。每轮比较a[i]并可能更新min，保持invariant。退出时i=a.Length表示已扫描全数组，结合invariant推出min≤所有a[j]。这三步（建立、保持、退出）构成完整证明链。

题目语境怎么用：Find/min题的关键是写清楚"min是已扫描前缀的最小值"——invariant须含范围约束(0<i≤a.Length)和语义约束(∀k∈[0,i): min≤a[k])。

卷面写法：写Precondition: a.Length≥1；Invariant: 1≤i≤a.Length ∧ ∀k·0≤k<i⇒min≤a[k] ∧ ∃k·0≤k<i∧min=a[k]；然后建立、保持、退出三段各一段说明。

常见失分点：只给postcondition而不展示invariant三步推理——无invariant则无证明。或invariant缺少范围约束导致∀k中k无界限。

标准答案（先写结论）：多证明 termination；在 Find loop 里可用 a.Length - i 作 variant，每次 i := i+1 后严格变小。

为什么这是对的：Partial correctness不排除死循环。Total加termination proof需一个variant——每轮严格递减且有下界的表达式。Find循环中a.Length-i每轮减1（i加1），下界为0（i≤a.Length），因此循环必然终止。variant利用"每轮处理一个元素，剩余元素数递减"的规律。

题目语境怎么用：线性扫描循环的variant通常是n-i或a.Length-j；二分查找的variant通常是right-left。

卷面写法：先完整写partial correctness（pre、post、invariant三步），再加Termination段："variant=a.Length-i，初始≥0，每轮i:=i+1后严格减少，当variant=0时循环退出，故必终止。"

常见失分点：选会增长的表达式作variant（如直接用i），或选不随循环变化的常量。

标准答案（先写结论）：Model checking，因为它要探索大量系统状态和路径。

为什么这是对的：Model checking穷举状态图所有路径。并发系统中n个进程各k个局部状态，组合全局状态数可达kⁿ——每增一个进程翻倍。如3进程各10态=1000，4进程=10000。SAT/SMT虽也面临大搜索空间，但其困难来自变量赋值组合爆炸，而非并发交错的"状态爆炸"——成因不同。

题目语境怎么用：涉及多进程协议或线程交互时，提到state explosion可关联到symbolic model checking（BDD压缩）、partial order reduction（去冗余交错）、abstraction（减状态细节）。

卷面写法：先定义状态爆炸（状态数随并发组件指数增长），说明它是model checking瓶颈，最后提至少一种缓解技术及原理。

常见失分点：把state explosion归因到SAT solver或Dafny——SAT面临组合搜索，Dafny用演绎推理（Hoare+Z3），都不靠穷举状态遍历。

标准答案（先写结论）：写 Balance == deposits.total - withdrawals.total，并要求 constructor/deposit/withdraw 前后保持。

为什么这是对的：BankAccount三字段（Balance、deposits、withdrawals）相互约束：Balance必须等于存款总和减取款总和。若无Valid()，deposit可只加钱不记录，导致不一致。Valid()集中表达此等式，constructor建立它（Balance=0且两序列空），deposit和withdraw各自维护它。Ghost意味着deposits/withdrawals编译后不存在——零运行时开销。

题目语境怎么用：对象有多个互相约束的字段→立即考虑写ghost predicate Valid()作为class invariant，在ensures中引用。

卷面写法：ghost predicate Valid() reads this { Balance == deposits.total - withdrawals.total }；constructor末尾加ensures Valid()；每个public method加ensures Valid()。

常见失分点：只写ensures Balance变化量而忘维护deposits/withdrawals记录和Valid()，导致后续调用时Valid()无法证明。

标准答案（先写结论）：用 G(request -> F response) 表示每次 request 后最终出现 response。

为什么这是对的：LTL的G表示"全局地、所有时刻"；F表示"最终"。G(request→F response)读作"永远地，若request成立则response最终成立"。若只用F(request∧response)只表示存在时刻两者同时成立，无法表达"每次request触发response"。若忘G写request→F response只检查初始状态——其他时刻的request被忽略。

题目语境怎么用：LTL活性的标准模板就是G(trigger→F outcome)；safety模板是G¬(bad)或G(condition)。

卷面写法：先写Promela/SMV模型的状态和transition，再写LTL性质。每个temporal operator后跟自然语言解释。

常见失分点：用LTL公式表达CTL语义（想写AG却只写G，无路径量词A），或忘G导致公式只在初始状态生效。

2. 逻辑：把英文要求变成公式

Predicate 是一个带参数的真假判断。比如 Student(x) 表示 x 是学生，Took(x, pen) 表示 x 拿了笔。Predicate logic 题就是把英文句子翻译成这些判断和量词。

符号 意思 读法
∀xfor all x所有 x 都...
∃xthere exists x存在至少一个 x...
P -> Qif P then Q如果 P 成立，则 Q 成立。
P ∧ QandP 和 Q 都成立。
P ∨ QorP 或 Q 至少一个成立。
¬Pnot PP 不成立。

Tautology / Satisfiable / Unsatisfiable

词 从零理解 例子
Tautology不管变量真假怎么取，公式总是真的。P ∨ ¬P
Satisfiable至少有一种取值能让公式为真。P ∧ Q 在 P=true, Q=true 时为真。
Unsatisfiable没有任何取值能让公式为真。P ∧ ¬P

内容理解题（先做 3 题）

核心答案（先确认概念）：Predicate 带变量；proposition 在给定变量后有确定真假。

为什么这是对的：Predicate本身不是命题——Student(x)取决于x是谁，不给x无法判断真假。绑定具体值（如Student(John)）后才变proposition。这个区别决定量词的作用：∀x Student(x)说"对所有x，Student(x)为真"，量词把predicate变成proposition。翻译英文时，先识别predicate（需参数），再识别连接词，最后根据"every"用∀、"some"用∃加量词。

题目语境怎么用：看到"every X does Y"，predicate是X(x)和Y(x)，量词∀，连接→。看到"some X does Y"，量词∃，连接∧。

卷面写法：第一步声明predicate名称和参数域（如Student(x): x is a student），第二步写公式∀x(Student(x)→Graduated(x))，第三步给实例说明。

常见失分点：∀搭配∧（如∀x(Student(x)∧Graduated(x))）——要求宇宙中每个东西都是学生且都毕业了，荒谬。∀要搭配→，∃要搭配∧。

核心答案（先确认概念）：没有任何赋值能让公式成立。

为什么这是对的：Unsatisfiable公式是"内在矛盾"——不管怎么赋值都是false。P∧¬P是最简例：P和¬P不可能同时为真。更复杂如(P∨Q)∧¬P∧¬Q自相矛盾。在CS603中unsatisfiable是验证的"好事"：要证F永远为真(valid)，只需证¬F是unsatisfiable——如果不存在使¬F为真的情况，F就在所有情况下为真。

题目语境怎么用：Z3验证：push; assert(not F); check-sat返回unsat说明F valid。SAT返回unsat表示公式无解。

卷面写法：区分三概念各给例子：tautology=总是真(P∨¬P)；satisfiable=有真赋值(P∧Q)；unsatisfiable=永假(P∧¬P)。

常见失分点：把satisfiable当成"所有赋值都真"——satisfiable只要求至少一种；把tautology和satisfiable混淆——tautology是更强的性质。

核心答案（先确认概念）：变量范围，例如数组下标必须限制在 0 ≤ i < a.Length。

为什么这是对的：没有范围的∀意味着"宇宙中所有东西"，但对程序来说变量只能取有效值。如"数组中所有元素≥0"写成∀x·a[x]≥0不限制x，公式含越界下标，无意义。正确写法是∀x·0≤x<a.Length⇒a[x]≥0。Dafny的invariant变量i必须写0≤i≤a.Length给verifier边界信息，否则报index out of range。

题目语境怎么用：数组量词先写∀k·0≤k<a.Length⇒...；集合量词写∀e·e∈S⇒...；整数范围写∀n·n≥0⇒...。

卷面写法：公式中每个量词变量附上范围限制，数组写0≤j<a.Length，person写Person(x)作guard。

常见失分点：只写∀k·a[k]≥0不限制k——Dafny报index out of range；∀和∃范围互换意思全变。

考试题目题（再做 5 题）

标准答案（先写结论）：例如 Right(John)、Took(x, PenOfJohn)，公式可写 Right(John) => ∃x Took(x, PenOfJohn)。

为什么这是对的：句子主语是John，predicate Right需参数。took his pen中his指回John，用常量PenOfJohn。"Somebody"→∃。→方向是Right→∃而非反过来。"his pen"用常量PenOfJohn而非Took(x,y)∧BelongsTo(y,John)，因为他的笔是确定对象——不引入额外ownership关系除非题面要求。

题目语境怎么用：英文"if...then"直接用→；"somebody/someone"用∃；"everybody/each"用∀。代词指代用常量。

卷面写法：先声明predicate：Right(p:Person)、Took(p:Person,o:Object)，常量John:Person、PenOfJohn:Object。公式：Right(John)→∃x:Person·Took(x,PenOfJohn)。

常见失分点："his pen"翻译过度复杂化成ownership关系；→方向写反成∃→Right，与原句相反。

标准答案（先写结论）：∀x (Person(x) => Walking(x) ∧ Talking(x))。

为什么这是对的："Every"→∀，"is walking and talking"→Walking∧Talking。关键结构是∀搭配→，而非∧。若写∀x(Person(x)∧Walking(x)∧Talking(x))意为"宇宙中每个东西都是人且在走路说话"——猫狗桌子也必须如此，荒谬。正确写法中Person(x)作为→前件起过滤：是人时才要求走路说话；非人的东西自动满足。

题目语境怎么用：口诀："∀ goes with →，∃ goes with ∧"。∀描述"所有某类对象都有某性质"，∃描述"存在某对象满足多条件"。

卷面写法：声明Person(x),Walking(x),Talking(x)。公式：∀x(Person(x)→Walking(x)∧Talking(x))。加注：对所有x，若x是人则x走路且说话。

常见失分点：∀后用∧代替→——predicate logic翻译最常见的top 1错误。漏Person(x) guard导致公式对非人实体也成立。

标准答案（先写结论）：要。题目明确要求 explain terms with example。

为什么这是对的：CS603解释题大概率要求"explain with example"，纯定义不得满分。Tautology举P∨¬P——无论P是T/F公式都为T。Satisfiable举P∧Q——P=T,Q=T时为T，但P=T,Q=F时为F，故非always true。Unsatisfiable举P∧¬P——P和¬P不可能同时为T。给具体赋值比只写定义强得多。

题目语境怎么用：带"explain with example"字样的题必须每个概念配例子。例子越简单越好——P∧¬P比复杂公式更清晰。

卷面写法：每概念先一句定义，再写公式例子，说明为何满足该分类（可给truth table或文字说明赋值情况）。

常见失分点：只给定义不给例子；或例子写错反而否定定义（如给P∨Q当tautology——它不是）。

标准答案（先写结论）：Ack 有两个递归参数，用 decreases m, n 按词典序证明递归调用整体变小。

为什么这是对的：Ackermann有两条递归：Ack(m-1,Ack(m,n-1))和Ack(m-1,1)。单参数不够——有时m不变n减，有时m减但n可能因内层Ack返回极大。Tuple decreases m,n用词典序：先比m，m小则整体小；m相等再比n。这覆盖所有递归路径，Dafny原生支持词典序验证。

题目语境怎么用：函数有≥2参数且不同分支递减不同参数→必须tuple decreases。常见：Ackermann、嵌套递归、多下标递归。

卷面写法：写decreases m,n，解释词典序规则（先比第一分量，相等再比第二），用一条递归路径例示(m,n)如何严格减小。

常见失分点：用decreases m+n或m*n——Ack某些路径算术表达式不一定单调减小；只写decreases m——n>0分支m不变。

标准答案（先写结论）：写 Balance == deposits.total - withdrawals.total，并要求 constructor/deposit/withdraw 前后保持。

为什么这是对的：BankAccount的核心invariant是"余额=存款总和-取款总和"。无此不变式，deposit可只加Balance不记录，导致数据不一致。ghost意味着deposits/withdrawals仅供证明用，编译后不存在。constructor建立（Balance=0且两序列空），deposit/withdraw分别维护等式。

题目语境怎么用：题目有对象且多字段有算术关系→写ghost predicate Valid()作class invariant。

卷面写法：ghost predicate Valid() reads this { Balance == deposits.total - withdrawals.total }，每个public method的ensures写Valid()。

常见失分点：忘写reads this；或mutating method忘在ensures中写Valid()，导致后续调用无法证明对象一致。

3. Hoare Logic：证明程序片段的前后条件

Hoare triple 写成 {P} S {Q}。意思是：如果程序片段 S 开始前 P 成立，并且程序正常结束，那么结束后 Q 成立。

概念 从零理解
Partial correctness如果程序结束，结果一定对；但不保证一定结束。
Total correctness结果对，并且程序一定结束。
Loop invariant循环开始前、每轮后、循环结束时都能用的事实。
Variant / decreases用来证明循环会结束的度量，每轮都变小且不能无限变小。

Loop proof 必写三件事

  1. Establishment：循环第一次检查 guard 前，invariant 已经成立。
  2. Preservation：假设 invariant 和 guard 成立，执行 loop body 后 invariant 仍成立。
  3. Exit reasoning：循环结束时，invariant 和 not guard 一起推出 postcondition。

Partial correctness answer shape: identify the precondition and postcondition; state the loop invariant; show it is established before the loop; show it is preserved by the loop body; when the loop exits, combine invariant with negation of guard to derive the postcondition.

Total correctness additionally needs a variant/decreases expression that is non-negative and strictly decreases on each loop iteration.

内容理解题（先做 3 题）

核心答案（先确认概念）：找到 invariant，并证明建立、保持、退出推出 postcondition。

为什么这是对的：循环体重复执行，无法像赋值语句"从后往前代"。Invariant是"归纳假设"：先证初始成立（base case），再证每轮保持（inductive step），最后退出时结合条件推出结论——与数学归纳法同构。如证n次迭代后sum=1+2+...+i，初始i=0,sum=0成立；每轮sum:=sum+i保持；退出i=n时sum=1+2+...+n。

题目语境怎么用：Find/min题设计为"min是已扫描前缀a[0..i)的最小值"；copy题设计为"b[0..i)已复制为a[0..i)"。

卷面写法：顺序写：(1)Precondition，(2)Invariant，(3)Establishment段，(4)Preservation段，(5)Exit+postcondition推导段。三段用自然语言加公式论证。

常见失分点：只写postcondition不写invariant——等于声称结论无证明。Invariant太弱撑不起postcondition——如只有i≤n没有语义条件。

核心答案（先确认概念）：多 termination proof，通常用 variant。

为什么这是对的：Partial允许循环forever但"如果终止就对"——可能是个死循环。Total要求variant：每轮严格递减且有下界（非负整数）。正整数不能无限递减，循环必终止。如while i<n do i:=i+1，variant为n-i（非负、每轮减1）。variant必须随循环推进减小，不能用常量或循环外变量。

题目语境怎么用：线性扫描用a.Length-i或n-k；值递减循环用变量本身如x（需证每轮x减小且非负）。

卷面写法：先完成partial全部内容，再加："Termination: define variant V=...；V≥0（证明）；每轮V严格减少（证明）；根据well-foundedness循环必终止。"

常见失分点：variant和invariant混淆——invariant维持不变，variant严格递减。或variant选错——选不随循环变量变化或会增长的量。

核心答案（先确认概念）：因为要知道赋值前需要什么条件，才能保证赋值后 postcondition 成立。

为什么这是对的：Hoare的assignment axiom是{Q[x:=E]} x:=E {Q}——要证赋值后Q成立，需赋值前满足"把E代入Q中x"的条件。如证{?} x:=x+1 {x>0}，把Q中x替为x+1得x+1>0即precondition。如果正着看反而难推导——postcondition是目标，assignment axiom从目标回推前提。

题目语境怎么用：遇x:=E时，把postcondition中的x替成E就是precondition。

卷面写法：用substitution notation写：wp(x:=E,Q)=Q[x:=E]（weakest precondition），说明代入过程。

常见失分点：代入方向反——应是"postcondition中旧x换成E"，非"E换成x"。或多赋值连续时漏中间代入步骤。

考试题目题（再做 5 题）

标准答案（先写结论）：已扫描部分 a[0..i) 中，min 不大于每个已扫描元素，并且 1≤i≤a.Length。

为什么这是对的：Find循环从i=1开始逐个扫描。初始i=1，已扫描a[0]一个元素，min=a[0]自然是最小值——invariant成立。每轮检查a[i]：若更小则更新min，否则保持——invariant保持。退出时i=a.Length，已扫描全数组，min即全数组最小值。范围1≤i≤a.Length保证数组有≥1元素且i不越界。

题目语境怎么用：Find/min的invariant模板："∀k·0≤k<i⇒min≤a[k]"加范围"1≤i≤a.Length"加"∃k·0≤k<i∧min=a[k]"确保min来自数组。

卷面写法：invariant 1≤i≤a.Length ∧ ∀j·0≤j<i⇒min≤a[j] ∧ ∃j·0≤j<i∧min=a[j]。然后论证Establishment、Preservation、Exit。

常见失分点：漏∃部分——无"min确实等于某数组元素"则verifier不知min来源；∀范围错写0≤j≤a.Length而非0≤j<i。

标准答案（先写结论）：结合 invariant 的已扫描范围，当 i==a.Length 时已扫描部分就是整个数组。

为什么这是对的：guard是i<a.Length，退出条件i≥a.Length。Invariant保i≤a.Length，两者交集得i=a.Length。此时invariant中∀j·0≤j<i变∀j·0≤j<a.Length——恰好覆盖全数组。invariant后半句"min≤a[j] for all已扫描j"直接升级为"min≤a[j] for all j in全数组"，即postcondition。

题目语境怎么用：任何扫描类循环（Find、Copy、LinearSearch）的退出推导同一模式：退出时i=边界，invariant中"已处理部分"等于"全部"。关键是invariant须含i上界使得退出能精确推出。

卷面写法："Exit: guard¬(i<a.Length)给i≥a.Length；invariant给i≤a.Length；因此i=a.Length。代入invariant的∀j·0≤j<a.Length⇒min≤a[j]，得postcondition。"

常见失分点：invariant中无上限i≤a.Length——退出时只知i≥a.Length，无法得i=a.Length，∀范围无确定上界。

标准答案（先写结论）：可选 a.Length - i，每次循环 i := i+1 后严格减少且非负。

为什么这是对的：循环体唯一赋值i:=i+1，依赖i递减的表达式可作variant。a.Length-i每轮减1，从a.Length-1递减到0。非负性由invariant中i≤a.Length保证。不能用i本身——i是递增的不满足"严格递减"；不能用min——不保证变化。variant选择原则：找循环体最确定的递增量，构造递减表达式。

题目语境怎么用：线性扫描循环variant总是"上限-循环变量"；逆序扫描可直接用循环变量本身（如果递减）。

卷面写法："Variant V=a.Length-i。初始=a.Length-1≥0。每轮i:=i+1后V变V-1，严格减小。V=0时i=a.Length，循环退出。故循环必终止。"

常见失分点：选i作variant——i每轮增加不满足递减。或选了但未证非负性——variant须下有界才保终止。

标准答案（先写结论）：用 a.Length - i；每次循环 i 增加，variant 严格变小且非负。

为什么这是对的：Find循环每轮i加1，剩余待处理元素数a.Length-i每轮减1。当剩余0时循环退出。variant非负由invariant中i≤a.Length保证——i永不超过数组长度。这个variant选择自然：循环扫描数组，每次前进一格，剩余格数即递减测度。

题目语境怎么用：任何"从0遍历到n"的循环variant模板是n-currentIndex。while逐次除2的情况variant是当前值本身。

卷面写法："Total correctness additionally requires a variant. Let T=a.Length-i. Initially i=1→T=a.Length-1≥0. Each iteration i:=i+1→T decreases by 1. Since T is non-negative and strictly decreases, loop must terminate."

常见失分点：variant初始值算错（如写a.Length而非a.Length-1）；未引用invariant证明variant非负。

标准答案（先写结论）：Ack 有两个递归参数，用 decreases m, n 按词典序证明递归调用整体变小。

为什么这是对的：Ackermann定义含两条递归：Ack(m-1,Ack(m,n-1))和Ack(m-1,1)。用decreases m——Ack(m,n-1)分支m不变，verifier报错。用decreases n——Ack(m-1,...)分支中内层Ack返回值可极大，n不减反增。Tuple用词典序：任何递归中要么m变小，要么m不变n变小——覆盖所有路径。

题目语境怎么用：多参数递归+不同分支递减不同参数=必须tuple decreases。常见：Ackermann、嵌套递归、多下标递归。

卷面写法：写decreases m,n，说明词典序定义——先较第一个，相等再较第二个。用一条递归路径例示(m,n)如何严格变小。

常见失分点：写decreases m+n——Ack中某些路径m+n不一定减小（内层Ack返回值可极大）。词典序tuple正确且安全。

4. Dafny：把证明写进代码旁边

Dafny 是一种带 verifier 的程序语言。你写方法时不仅写代码，还写 requires、ensures、invariant、decreases。Verifier 会尝试证明：所有满足 requires 的输入，执行后都满足 ensures。

Dafny 关键词 人话解释 考试中常问
requires调用者必须保证的条件。缺少 requires 会导致方法无法安全使用数组。
ensures方法承诺结束后成立的条件。说明返回值满足什么性质。
invariant循环中一直保持真的条件。数组循环、binary search、copy method。
decreases证明递归或循环会停止的度量。Ackermann、fib、while loop termination。
assert要求 verifier 在该位置证明某事实。解释为什么 assert 可/不可验证。
ghost只用于证明的代码，编译后删除。ghost field、ghost predicate Valid。

Dafny loop invariant 怎么找

Invariant 通常是"到目前为止已经处理的部分是正确的"。如果循环变量是 i，常见 invariant 会包含范围 0 <= i <= a.Length，再加上已经处理过的区域性质，例如 forall k :: 0 <= k < i ==> ...。

Class invariant 和 ghost predicate

Class invariant 是对象任何公开方法结束后都应该保持的规则。Dafny 常用 ghost predicate Valid() 表达这个规则。Ghost 的意义是：证明时需要它，但运行时程序不需要它。

解释 class verification 时可写：the constructor must establish the class invariant; each public method may assume the invariant at entry and must re-establish it before exit; ghost fields and predicates help express abstract state and validity conditions without affecting compiled code.

内容理解题（先做 3 题）

核心答案（先确认概念）：不会自动知道循环整体做了什么，需要 invariant 描述已处理部分和范围。

为什么这是对的：Dafny verifier本质是定理证明器，遇while循环只读单次迭代——不会"猜测"循环整体效果。若写求和循环不加invariant，verifier只知循环前后状态，不知每轮sum和i的关系。必须写invariant告诉verifier"每轮后sum=前i元素之和"。同样数组访问靠invariant告知i在合法范围内，否则可能报index out of range。

题目语境怎么用：看到循环第一反应：需要invariant描述(1)循环变量范围，(2)已处理部分与未处理部分的关系。Dafny loop题核心就是设计正确且足够的invariant。

卷面写法：写invariant 0≤i≤a.Length（边界），再加invariant ∀k·0≤k<i⇒...（语义条件），解释verifier如何用这些invariant完成证明。

常见失分点：只写边界invariant不写语义invariant——verifier定i不越界但不知循环做了什么有用事，ensures无法推出。

核心答案（先确认概念）：证明递归或循环会终止，避免 verifier 无法确认 termination。

为什么这是对的：Dafny默认要求所有循环和递归终止（total correctness为默认）。若写while true或递归函数无decreases且verifier找不到自然递减参数，报"cannot prove termination"。对递归，decreases标注哪个参数在每次递归调用中变小。对循环，Dafny自动尝试找variant，找不到时需显式提供decreases clause。

题目语境怎么用：递归函数第一个注解写decreases；循环若Dafny自动推断不了（如while true内嵌break），需手动加decreases。

卷面写法：写decreases n或decreases m,n（词典序），解释为何每次递归/迭代时decreases表达式严格减小。

常见失分点：decreases选的表达式实际不递减（如选了递归中没变的参数），或需tuple却只写单个参数。

核心答案（先确认概念）：Constructor 建立它，所有会改变状态的方法都要保持它。

为什么这是对的：Dafny class的Valid()在所有公开可见点都必须成立。Constructor创建对象时须确保Valid()为真（initial establishment）。此后每个public method进入可假设Valid()成立（因上方法结束时已维护），执行中间可暂时打破Valid()——只要退出前恢复。Method body中间段临时不一致不暴露给外部。这就是Dafny class verification核心契约：constructor establishes, methods preserve。

题目语境怎么用：BankAccount的Valid()是Balance==deposits.total-withdrawals.total；constructor后Balance=0两序列空等式成立；deposit操作后需确保等式仍成立。

卷面写法："Constructor must establish Valid() at exit. Each public method assumes Valid() on entry and must re-establish Valid() before exit."

常见失分点：只写constructor建立Valid()忘在mutating method的ensures中重写Valid()——导致chain断，后续方法无法假设Valid()。

考试题目题（再做 5 题）

标准答案（先写结论）：Ack 递归有两个参数，tuple metric 允许按词典序证明递归调用整体变小。

为什么这是对的：Ackermann m=0时返回n+1（终止）；否则m>0时分两路递归。某些路径减m（n可能增大），某些减n（m不变），单参度量会被另一分支反例击穿。Tupledecreases m,n用词典序：先比m，m减则整体减；m等再比n，n减则整体减。Dafny原生支持词典序验证，覆盖所有路径。

题目语境怎么用：三问判断需tuple：(1)函数有≥2参数？(2)不同分支递减不同参数？(3)一参减小时另一可增大？有一Yes就考虑tuple。

卷面写法：写decreases m,n，分情况说明：m>0∧n=0时m减；m>0∧n>0时先内层m,n-1减n，外层m-1减m——两条路径词典序都保证严格减小。

常见失分点：用decreases m+n或m*n——某些递归路径算术表达式不一定单调减小，词典序tuple标准且安全。

标准答案（先写结论）：题目要求修复/解释循环正确性和终止性，invariant 证明结果关系，variant 证明循环停止。

为什么这是对的：Dafny题"找错"不是只改语义bug——verifier报错本质是"annotation不足"或"proof obligation无法discharge"。循环缺invariant→verifier不知循环做什么，ensures无法成立；缺variant→verifier无法证终止。"修复"意味着补充正确的invariant（描述循环每轮维护的不变关系）和variant（证循环不无限执行）。

题目语境怎么用：Dafny code题标准作答流程：(1)识别code errors（如缺requires致数组越界）；(2)写缺的annotation；(3)解释每个annotation为何足够。

卷面写法：先指错："The loop lacks an invariant describing the relationship between x and y after each iteration."再补写invariant和variant，解释invariant如何从loop body保持，variant为何递减。

常见失分点：只写代码修改不写解释——Dafny题要求"why the annotation is sufficient"，非只贴代码片段。

标准答案（先写结论）：把对象一致性条件集中成可复用证明义务，方法前后都要维护它。

为什么这是对的：BankAccount三字段(Balance、deposits、withdrawals)须满足"Balance=deposits总和-withdrawals总和"。若不写Valid()，每个method的ensures都要重复此等式；若将来加新约束(Balance≥0)，须改所有method。Valid()集中所有约束到一个predicate，每个method只需ensures Valid()一次。Ghost意义：编译后deposits、withdrawals和Valid()不存在——零运行时开销。

题目语境怎么用：题目出现class带多互相约束的字段→第一时间考虑ghost predicate Valid()作class invariant。

卷面写法：解释三句：(1)Valid()表达对象一致性条件；(2)Ghost表示证明专用不编译；(3)Constructor establishes it, every public method preserves it。

常见失分点：只解释ghost是什么而不解释Valid()为何重要——需同时覆盖"一致性表达"和"ghost用途"两层。

标准答案（先写结论）：在已扫描区间 a[0..i) 内，min 不大于每个已扫描元素，并保持 1<=i<=a.Length。

为什么这是对的：Find invariant分两层。结构层：i范围1≤i≤a.Length——verifier需此保a[i]访问安全。语义层：∀k·0≤k<i⇒min≤a[k]且∃k·0≤k<i∧min=a[k]——前句说min不大于任何已扫描元素，后句说min确来自数组（防trivial solution如min取负无穷）。三层合并恰好：退出时i=a.Length，第一句变postcondition。

题目语境怎么用：Dafny invariant固定套路——先边界（i范围），再∀关系（已处理区域性质），最后检查退出时是否推出postcondition。

卷面写法：invariant 1≤i≤a.Length；invariant ∀k·0≤k<i⇒min≤a[k]；invariant ∃k·0≤k<i∧min=a[k]。每行注释用途。

常见失分点：只写∀不写∃——无∃则verifier不知min的"来源"，退出时无法得有意义结论（min可是任意小常数）。

标准答案（先写结论）：用 a.Length - i；每次循环 i 增加，variant 严格变小且非负。

为什么这是对的：Find的递增量是i，递减量自然是a.Length-i。初始i=1，variant=a.Length-1（数组长≥1时≥0）。每轮i:=i+1后variant减1。variant=0时i=a.Length导致while guard为false，退出。非负性由invariant中i≤a.Length保证。Dafny中decreases a.Length-i即Hoare的variant。

题目语境怎么用：while循环的Dafny decreases从句=Hoare logic的variant，写法和证明完全对应。

卷面写法：while关键字后写decreases a.Length-i，说明"Initially≥0, each iteration decreases by 1, eventually reaches 0 forcing loop exit."

常见失分点：Dafny中decreases *表示"不检查终止"——考试中极少用，正常都提供具体decreases表达式。

5. Model Checking：穷尽检查状态系统

Model checking 用在 reactive systems，也就是不断和环境交互的系统，例如协议、并发程序、控制器。它把系统建成有限状态图，然后自动检查 temporal logic 公式是否成立。

Kripke structure一个带状态、转移、初始状态和标签的状态图。
Atomic proposition在某个 state 中为真或假的简单命题。
Path沿着 transition 走出来的一条状态序列。

LTL 和 CTL

Temporal logic 用来表达"将来会发生什么"。LTL 看一条线性时间路径；CTL 可以明确说"所有路径"或"存在某条路径"。

Spin / NuSMV / Runtime Verification

Spin 使用 Promela 建模，常用于 LTL 和并发系统。NuSMV 使用 SMV 建模，支持 CTL 和 LTL。Model checking 在运行前检查模型；runtime verification 在系统运行时监控实际执行轨迹，因此通常只能看到实际发生的路径。

内容理解题（先做 3 题）

核心答案（先确认概念）：适合状态系统、协议、并发或时间性质，而不只是单个函数前后条件。

6. SAT、SMT、Z3 和 DPLL

SAT 问的是：有没有一种 true/false 赋值让布尔公式为真？SMT 是 SAT 的升级版，不只处理布尔变量，还能处理整数、数组、函数等理论。Z3 是一个 SMT solver。

词 从零理解
SAT找布尔公式的 satisfying assignment。
CNFConjunctive Normal Form，很多 clause 用 AND 连起来，每个 clause 里是 OR。
DPLL经典 SAT 搜索算法：尝试赋值、传播约束、遇到冲突就回溯。
CDCLConflict-Driven Clause Learning，从冲突中学习新 clause，避免重复犯错。
SMTSAT + 背景理论，例如整数算术、数组、未解释函数。
Z3微软开发的 SMT solver，可返回 sat/unsat/model。

Validity 和 satisfiability 的关系：要证明公式 F 永远为真，可以让 Z3 检查 not F 是否 unsatisfiable。如果 not F 没有任何模型，那么 F 就是 valid。

内容理解题（先做 3 题）

核心答案（先确认概念）：多个 clauses 的 AND，每个 clause 是多个 literals 的 OR。

为什么这是对的：CNF标准化SAT solver输入。任何命题逻辑公式可转换CNF。例如(P→Q)∧R消→得(¬P∨Q)∧R——已是CNF：两clause用AND连。若公式非CNF，solver需先做Tseitin变换引辅助变量。DPLL和CDCL假设输入为CNF——通过unit propagation简化clause（某clause只剩一literal则该literal必为真），搜索中学到的conflict clause也直接加入CNF集。

题目语境怎么用：SAT题给出公式后先检查是否CNF——可快速判断哪些是unit clause、哪些是pure literal，手算DPLL。

卷面写法：给例子标注：(A∨¬B∨C)∧(¬A∨D)∧(B∨¬C)——3个clause，每个clause是OR of literals，整体AND。

常见失分点：把clause内部当成AND——错误如"CNF是(A∧B)∨(C∧D)"，这实际是DNF(Disjunctive Normal Form)，与CNF对偶。

核心答案（先确认概念）：系统地猜变量真假、简化、发现冲突后回溯。

为什么这是对的：DPLL是"带智能剪枝的穷举"。纯穷举：n变量试2^n种赋值，太慢。DPLL每步选一未赋值变量，先假设为真（分支），然后unit propagation推导所有被迫赋值——如果某clause除一literal外全假，那literal必为真。若propagation致冲突（某clause全假），回溯到决策点翻转假设。另有pure literal elimination——若某变量在所有未满足clause中只以同一极性出现，可直接赋该值。

题目语境怎么用：手算DPLL题按步骤：(1)看有无unit clause→unit propagate；(2)看有无pure literal→pure literal assign；(3)都无就选变量split。每步标受影响clause简化结果。

卷面写法：给CNF例子(如(P∨Q)∧(¬P)∧(¬Q∨R))，逐步展示：unit propagate¬P→P=F→clause1剩Q→Q=T→clause3未知→选R=T→sat，给model。

常见失分点：跳过unit propagation直接split——DPLL力量在于unit propagation可减少大量分支，漏它等于回穷举。或split后忘回溯翻转赋值。

核心答案（先确认概念）：程序验证常产生整数、数组、等式等约束，SMT solver 能处理这些理论。

为什么这是对的：程序验证产生的proof obligation很少是纯布尔——大部分是"x加1后是否大于y"这种整数算术、"数组a第k元素是否等于某值"这种数组理论、"两指针是否指向同对象"这种未解释函数。纯SAT solver不懂整数数组，须把数字展成二进制(bit-blasting)效率极低。SMT solver(如Z3)内置整数算术(LIA/LRA)、数组(select/store)、UF等决策过程，在SAT引擎上用Nelson-Oppen组合各理论求解——这正是Dafny verifier底层引擎。

题目语境怎么用：Z3 lab类题中分清楚哪些是命题约束(SAT能解)，哪些涉整数/数组(需SMT)。Dafny每题背后都是Z3做SMT solving。

卷面写法：先说SAT只处理布尔；再提程序验证典型constraint含整数算术、数组存取等；总结SMT=SAT+theories，例举Z3处理bool+int约束。

常见失分点：说SMT是"更快的SAT"不提theories——SMT关键升级是能理解非布尔domain，非只更快。

7. FRET、Data Refinement 和 Under-specification

FRET 帮助把自然语言需求变成更形式化的需求，方便后续生成或检查 temporal formulae。Data refinement 是用更具体的数据结构实现更抽象的规格，只要外部可观察行为保持一致即可。

概念 人话解释 考试答法
Under-specification规格说得太少，允许很多不想要的实现也通过。解释缺少约束，并补充 contract/invariant。
Data refinement抽象模型换成具体实现，但保持用户看到的行为。例如 abstract set 用 array/list 实现。
FRET把自然语言需求结构化，减少歧义，并连接到形式化检查。提到 requirements documentation、formalisation、model checking。

内容理解题（先做 3 题）

核心答案（先确认概念）：需求没说清时，多个实现都可能自称正确，但行为不符合用户真实期待。

为什么这是对的：Under-specification即需求"留白太多"。如"排序后输出有序数组"——未说输入是否被修改、是否stable、重复元素如何处理。一个实现可返新数组(overhead大)，另一个可原地修改(side-effect)，还有可丢重复元素——都满足"输出有序"，但调用者预期完全不同。在Dafny中表现为缺requires/ensures或invariant不够强——verifier不能说"错"也不能保证"对"。补全法：在contract加约束，如ensures元素multiset相等。

题目语境怎么用：考试给一段需求描述，让你指哪里under-specified，写出补充的contract或invariant。

卷面写法：(1)指under-specification具体点——哪个行为未被约束；(2)解释现有spec不够——举例两不同行为都满足spec；(3)给补充constraint，写成Dafny requires/ensures或LTL公式形式。

常见失分点：只说"spec不够"不给出具体缺失什么、不写补充约束——无具体内容的批评不得分。

核心答案（先确认概念）：把自然语言需求拆成触发、条件、响应和时间范围，减少歧义。

为什么这是对的：自然语言需求如"系统应快速响应"有三问题：(1)歧义——"快速"0.1s还是5s？(2)不完整——触发条件是什么？(3)不可验证——"快速"不可测量。FRET把每条需求拆成标准六元组——Condition(条件)、Trigger(触发)、Response(响应)、Timing(时限)、Scope(范围)、Constraint(约束)——从此结构化描述自动生成LTL/FRET-ish公式供model checker验证。FRET本质是"需求结构化的中间语言"。

题目语境怎么用：题目给自然语言需求让你"formalise with FRET"——先拆Condition/Trigger/Response/Timing，再写semi-formal模板"Upon [trigger] when [condition] the system shall [response] within [timing]."

卷面写法：先列FRET分解字段（condition,trigger,response,timing），说明此结构化如何减歧义、如何衔接LTL formula generation。

常见失分点：把FRET当model checker或refinement technique——FRET是需求结构化工具，非验证工具也非实现技术。

核心答案（先确认概念）：具体表示的数据行为要保持抽象模型承诺的性质。

为什么这是对的：Data refinement出发点：有抽象模型(如用数学集合描述集合管理器)，然后用具体数据结构实现(如数组或链表)。关键要求：具体实现的每个操作对外表现须与抽象模型一致——叫"operation correspondence"(simulation)。例如抽象add把元素加入set S；具体add append到数组a并标记——然后prove coupling invariant如"set of elements in a = S"。若coupling invariant成立且每个具体操作能模拟抽象操作，具体实现refines抽象模型。

题目语境怎么用：Event-B或Dafny中遇abstract model→concrete implementation对应关系，核心是(1)coupling invariant和(2)operation refinement proof。

卷面写法：定义coupling invariant:I(abs,conc)连接抽象和具体状态。证每个具体操作：if I holds before and concrete op runs, then I holds after and abstract op's effect is matched.

常见失分点：只说"具体实现要正确"不提coupling invariant——无coupling invariant的refinement只是两独立程序，无法证一致。

为什么这是对的：SAT/SMT答案四必检：(1)概念定义——tautology/satisfiable/unsatisfiable区分正确且各举例子？(2)算法解释——DPLL/CDCL描述有步骤或例子？(3)SMT特殊性——提到theories(整数算术、数组、UF)而不只说"更快"？(4)Model——解释sat时为什么看model、unsat时为什么看unsat core？这四检查点覆盖了SAT/SMT的核心理解层次。

题目语境怎么用：Z3 lab题检查点：check-sat→sat? give model; unsat? explain why; valid? prove via negation unsat。

卷面写法：四个概念各给定义+例子；一个算法给逐步演示；SMT说theories；model说调试用途。

常见失分点：不区分sat/valid/tautology三个容易混淆的概念——sat是存在模型，valid是所有模型，tautology是命题逻辑中的valid。

为什么这是对的：Find Hoare题三项检查：(1)Precondition——a.Length≥1对吗？数组为空时程序会崩溃吗？(2)Invariant——含1≤i≤a.Length和∀+∃语义吗？三步论证每步有公式吗？退出时i=a.Length推出postcondition吗？(3)Variant(若total)——a.Length-i非负且每轮递减吗？引用invariant证非负了吗？三项任何一项缺论证，证明链即断。

为什么这是对的：Ack Dafny题两项检查：(1)Decreases选择正确吗？——是tuple(m,n)而非单参或算术表达式？(2)解释充分吗？——说明了词典序规则，且分递归路径论证每条路径下tuple严格减小？常见的扣分点是只写decreases m,n不解释词典序如何工作、不证每条路径递减。写"lexicographic ordering: compare first then second; path (a) m decreases; path (b) inner call n decreases, outer call m decreases"才算完整。

标准答案（先写结论）：是否用一个冲突学习例子解释 CDCL，而不是只展开缩写。

为什么这是对的：CDCL题的核心检查：答案里有具体conflict→learned clause的例子吗？纯定义"Conflict-Driven Clause Learning从冲突中学习子句"只得一半分——要展示一个mini实例。例如：公式含clauses (¬A∨B)∧(¬B∨C)∧(A)∧(¬C)。DPLL决定A=T→B=T(unit)→C=T(unit)→conflict(¬C假)。CDCL从这次冲突分析出learned clause (¬A)，永久记录"A不能为T"。这个具体链展示了learning如何剪掉整个搜索子树。

题目语境怎么用：SAT/SMT题凡是含"explain"或"example"字样的，必须给一个从头到尾的具体实例。自己准备一个3-4变量的通用CNF例子。

卷面写法：定义+例子格式："CDCL extends DPLL by learning from conflicts. Example: ... (show conflict→analysis→learned clause). This prevents the solver from revisiting the same conflicting subspace."

常见失分点：只展开CDCL缩写不举例——这和只写"CDCL=Conflict-Driven Clause Learning"得零分一样。

标准答案（先写结论）：Ack 有两个递归参数，用 decreases m, n 按词典序证明递归调用整体变小。

为什么这是对的：Ack的递归在m=0时基值(n+1)返回，否则m>0时分两路：m-1减m或n-1减n。单参不够——减m的路n可能增大，减n的路m可能不变。Tuple用词典序：任一路径(m,n)在词典序下严格减小。举例：若当前(m,n)=(2,1)，调用Ack(1,Ack(2,0))→内层(2,0)→(1,1)(因Ack(2,0)=1)→再外层(1,1)→各步(2,1)>(2,0)>(1,1)，每步或首分量减或首分量等次分量减。
