复习系统怎么用

目标不是收藏很多材料，而是让每一次阅读都能落到考试动作上。四门课都按同一种节奏走：从零理解，马上接小题，再落到真题模板。

1. 用线性页铺台阶

每个新术语第一次出现时先解释，再给最小例子。适合从教材讲义顺序一路学完单科。

2. 用题型页压答案

把知识点转换成考试表格、证明模板、代码分析步骤和检查清单。适合做题和考前复盘。

3. 用关键词补薄弱点

不会哪个词就搜哪个词，例如 HALT、BVA、Dafny、MVC，直接跳到相关小节。

4. 用模拟战押题型

每门课独立一页：预测几乎必出 → 很有可能 → 如果换题型会换什么。每道预测题附带可直接背诵的标准答案（含评分要点）。考前最后一天的最佳读物。

过关问答：先内容理解，再 past/sample paper 题，悬停或点击显示答案

内容理解题（先做 3 题）

核心答案（先确认概念）：先用单科教材线性页把概念学顺，再用题型页练考试答案，最后用关键词跳转补薄弱点。

为什么这是对的：线性页按知识依赖顺序编排——比如CS605从regular language到CFL再到NP，跳过前面直接看NP会完全看不懂术语。题型页的答案模板建立在概念理解之上，顺序颠倒等于背答案模板而不懂原理，换一道题就不会写。关键词跳转是点对点工具，只能补局部缺口，不能替代系统的学习路径。三步之间存在严格的依赖关系：没铺概念台阶就看不懂题型，没做过内容题就不知道哪里薄弱。

题目语境怎么用：打开任一科目时，先确认线性页上本节的概念是否都能用自己的话复述，再过到题型页看同一概念在考试中怎么出题。比如学完CS608线性页上的BVA定义和边界值选取规则后，立刻切到题型页看Climate.determine那道题里BVA表格有几列、每列填什么、expected result怎么从spec推导。

卷面写法：回答此类"使用顺序"题时，按"第一步→为什么不能跳过→第二步→依赖关系→第三步→适用范围"的结构写，每步各给一个具体例子。只说"先A再B再C"不加理由会丢解释分。

常见失分点：只列出三步但不解释依赖关系。评分标准要求说清"为什么是这个顺序"而不只是"是什么顺序"。另一个常见问题是跳到关键词搜索之前没确认自己概念真懂了——搜到一个术语的定义不等于理解了它和前后概念的关系。

核心答案（先确认概念）：线性页负责从零学习和铺台阶；题型页负责把同一批知识压成可交卷的模板、表格、证明或比较结构。

为什么这是对的：同一门课的知识在两页上服务于完全不同的目标。线性页像教材——先定义术语、给最小例子、再逐步深入，目的是"学懂"；题型页像参考答案汇编——直接把知识映射到past paper里出现过的题型和答案结构，目的是"能写"。两者读同一批知识但组织逻辑完全不同：线性页按概念依赖排序（先学Turing Machine再学HALT），题型页按考试题型分组（Q1黑盒、Q2白盒、Q3类内测试）。

题目语境怎么用：复习BVA时，在CS608线性页上学"什么是边界值、为什么边界容易出错、just below/on/above怎么选"，然后切到CS608题型页看"Climate.determine那道题的BVA表格有几列、TCI怎么编号、expected result从specification的哪句话推导"。线性页告诉你"是什么和为什么"，题型页告诉你"考试时怎么写答案"。

卷面写法：用对比结构回答：先说两页各自的目标（学懂 vs 能写），再说内容组织的区别（知识依赖顺序 vs 题型分类），最后给一个跨页对照的具体例子（如BVA在线性页的定义和题型页的表格形式）。

常见失分点：把题型页当教材从头读——题型页不解释概念，只在概念已知的前提下展示答案写法。没学线性页直接看题型页，会抄答案模板但换一道题就不会套。反过来，只看线性页从不看题型页，考试时知道概念但不知道答案该写成什么格式。

核心答案（先确认概念）：关键词跳转适合查漏补缺。基础没铺好时直接跳词，容易只记片段，不能形成考试答案。

为什么这是对的：关键词搜索只把你带到某个术语出现的位置，不提供上下文。搜"HALT"能看到归约的四步写法，但如果你不知道Turing Machine的基本定义和decidability的框架，你根本看不懂为什么那四步能导出矛盾。反过来，完整走过线性页+题型页之后，遇到的具体卡点往往只是一个术语或一个题型变体，关键词跳转正好精准命中——5分钟解决后继续推进。放在最后不是因为不重要，而是因为它作为"补丁工具"的效率依赖于"底层知识框架"已经建好。

题目语境怎么用：做past paper时卡在"short-circuit"这个词——不要回头重读整章，直接搜关键词跳到white-box coverage那节看short-circuit的bytecode-level解释，5分钟解决后继续做题。但如果卡在"整个branch coverage都不懂"，关键词解决不了——题目可能搜不到一个叫"branch coverage完全不懂怎么办"的条目，此时需要回到线性页从头补。

卷面写法：先定义关键词跳转的定位（补洞工具而非学习入口），再说它适用的场景（局部卡点、已学过但遗忘的术语）和不适用场景（系统性不理解、整章空白），最后给正反两个例子。

常见失分点：把关键词跳转当成学习入口使用——正确的入口永远是线性页，关键词只应该在"已经学过但忘了"或"做某道题时碰到一个陌生术语"时使用。考前焦虑时疯狂搜关键词以为搜得越多复习越充分，实际上碎片化搜索反而破坏知识结构。

考试题目题（再做 5 题）

标准答案（先写结论）：先看 CS608 线性页黑盒/BVA，再看 CS608 Q1 样章 worked BVA。

为什么这是对的：Climate.determine是一道典型的BVA题，考点不是会不会写代码，而是会不会从specification里提取阈值(16和60)并围绕每个阈值构造just below/on/above三组值。线性页教你BVA的理论——为什么边界容易出错、阈值怎么从spec中识别、边界值选取的规则；Q1样章给你看一个完整的worked example——TCI表怎么列、selected value怎么选、expected result怎么从spec推导。两者结合才能独立完成Climate那道题：理论告诉你"为什么这样选"，样例告诉你"卷面上怎么呈现"。

题目语境怎么用：Climate 题围绕 temp=16、humidity=60；其他 BVA 题也要先找 threshold、minimum、maximum 或 between 的边界。

卷面写法：卷面用表格写 TCI、selected value、test case、expected result。每个 expected result 都必须来自 specification。

常见失分点：不要只写"测边界"。只列输入没有 expected result 也不完整；题目说不含 error values 时，别把无效值混入有效边界表。

标准答案（先写结论）：先补 CS615 线性页的 3-tier/MVC/database，再到题型页 sample-topics 做架构比较。

为什么这是对的：FinTrack是一道架构设计题，不是选择题也不是填空题。你要从题面描述的系统需求（real-time price pushes、用户量、数据一致性要求）出发，推导出哪层负责什么、数据怎么存、瓶颈在哪里、trade-off是什么。线性页教你3-tier每层的职责和MVC的请求流——谁接request、谁执行业务规则、谁保存数据；sample-topics给你看microservices vs monolith、SQL vs NoSQL的比较框架。只有先理解"每层干什么"，才能根据题面信息做出有说服力的架构决策，而不是背诵"microservices更scalable"这种空话。

题目语境怎么用：把题面放回 browser -> route/controller -> service/model -> database -> response。若题面是 FinTrack real-time price，就强调缓存/streaming/polling、吞吐和前端刷新；若是 MVC/Rails/React/HTTP，就强调职责边界和请求流。

卷面写法：卷面先写结论，再贴题解释职责和 trade-off。例如 microservices 可单独扩展某个服务，但增加网络调用、部署、监控和数据一致性复杂度；monolith 简单但扩展边界较粗。

常见失分点：不要只写"scalable"或"handles logic"。必须说明是哪种数据压力、职责边界或失败风险让这个选择合理。

标准答案（先写结论）：先补 CS605 的 certificate/verifier，再补 NP-complete reduction from 3-SAT。

为什么这是对的：NP证明题有两个子任务，缺一不可。先证明问题in NP——给出certificate（CARA clique里是一个顶点子集）和polynomial-time verifier（检查子集大小是否为k且所有顶点两两有边，循环O(k²)即多项式时间）。再证明NP-hard——从已知NP-complete问题（3-SAT）构造reduction，把clause/literal映射到图结构，并证明原公式可满足iff目标图有大小为k的clique。先补certificate/verifier是为了建立"什么叫in NP"的肌肉记忆，再补3-SAT reduction是为了学会"怎么把一个已知难问题变形成目标问题"的标准套路。

题目语境怎么用：CARA clique、vertex cover、3-SAT clauses/literals 都是在给 certificate 和 verifier 的语境：clique 检查点集两两有 edge；3-SAT reduction 检查 clause/literal 到目标结构的对应。

卷面写法：in NP 写 certificate、verifier、polynomial time。NP-complete 写 membership，再从已知 NP-complete 问题构造并证明 yes iff yes。

常见失分点：不要把 NP-complete reduction 写成 HALT reduction。3-SAT reduction 用来证明 NP-hard；HALT reduction 用来证明 undecidable。

标准答案（先写结论）：搜 JaCoCo、branch coverage、short-circuit，再回到 Q2 题型页写 additional test + expected result + coverage explanation。

为什么这是对的：decideWrite是一道白盒覆盖率题。JaCoCo报告里的红色行表示statement从未执行——你需要构造一个输入让程序走到那行；黄色标记表示branch条件只覆盖了一侧——你需要构造输入让条件走向未覆盖的分支。short-circuit运算符(&&, ||)在字节码层面会产生额外的分支判断，JaCoCo会把它们一并标记出来，如果不知道这个机制就会困惑"为什么一个if有两个黄色标记"。理解了这三个概念，再看Q2题型页的worked example就能独立完成decideWrite的覆盖率补全。

题目语境怎么用：看到 line red/yellow，要问什么输入能到达这条 statement 或走到缺失 branch；short-circuit 条件还可能产生 bytecode-level branch。

卷面写法：卷面写 coverage item、input/call、expected result、补上的 line/branch。不要只给 input。

常见失分点：覆盖率不是正确性的替代品。每个新测试仍要有 specification 推出的 expected result。

标准答案（先写结论）：Sample A Q3 exception 21 和 Sample A Q5 line number n never executed，二者都要写 HALT mapping reduction。

为什么这是对的：HALT归约题的通用逻辑是——假设存在判定目标性质的程序D，构造程序N先模拟M(w)：如果M(w)停机则N表现出目标性质（如抛exception 21、执行第n行），如果M(w)不停机则N不表现该性质。于是D(N)返回true当且仅当M(w)停机，与HALT的不可判定性矛盾。练这两道题的原因：Q3练习"停机时触发exception"模式，Q5练习"停机时执行指定代码行"模式，覆盖了最常见两种需要归约到HALT的性质类型。两道练完后你就掌握了"把任意程序性质变成停机开关"的构造思路。

题目语境怎么用：构造程序 N 时通常先模拟 M(w)：若停机，就触发 exception、执行指定 line 或破坏某个 throughout 条件；若不停机，就不触发该性质。

卷面写法：卷面四步：假设 decider D 存在；从 <M,w> 有效构造 N；运行 D(N)；证明 M(w) halts iff N 具有题目性质；最后写 contradiction。

常见失分点：不能只写 reduce to HALT。mapping 怎么造、为什么 computable、iff 两个方向都要写。

每一节的过关闭环

流程最大的风险不是"没看材料"，而是看完就跳走，没有确认自己能不能说出来、写到卷子上。现在每个 section 底部都会出现 QA 和过关勾选，按下面四步走。

1. 读本节正文

只读到能用自己的话复述本节概念、例子和常见坑，不追求一次背完所有文字。

2. 做 3 道内容题

先遮住答案答内容理解题。能答出定义、关系、最小例子，才算内容地基过关。

过关问答：先内容理解，再 past/sample paper 题，悬停或点击显示答案

内容理解题（先做 3 题）

核心答案（先确认概念）：先用自己的话复述本节标题，再回到表格或模板里找关键词，确认自己知道它在考试中解决哪类问题。

为什么这是对的：复习最大的敌人不是"没看过"，而是"看过就以为自己会了"。复述本节标题强制你做主动回忆——如果你说不出BVA的三个关键点（阈值识别、just below/on/above选取、expected result推导），说明你只是眼熟而非真懂。回到表格或模板里找关键词是在做"概念→考试输出"的映射训练——你知道Hoare logic的定义不算过关，你知道卷面上要写precondition、invariant的三步验证、postcondition才算。判断标准必须同时检验理解和输出两个维度。

题目语境怎么用：学完CS608黑盒测试那节后，遮住屏幕问自己："黑盒测试有哪几种技术？BVA的边界值怎么选？TCI表有哪几列？"如果30秒内说不完整，回到正文重读。学完CS603 Hoare logic那节后，遮屏说出："Hoare triple的结构、Find-min的invariant是什么、invariant验证三步各检查什么"。

卷面写法：回答"闭合判断"类问题用"自测方法→判断标准（理解+输出两个维度）→不通过时的回退动作"三段结构，每段给一个本节的具体例子，不要写抽象的"能复述定义"而要写"能说出BVA的阈值识别规则和三个测试值的选取方法"。

常见失分点：用"感觉懂了"代替实际复述。大脑会欺骗你——看着屏幕觉得都认识，遮住屏幕一个字都说不出来。必须实际说出来或写下来才算数，不能跳过这个主动提取的步骤。

核心答案（先确认概念）：落到题目场景、证明步骤、测试表格、代码结构或 trade-off。只背定义通常不够。

为什么这是对的：考试不考默写定义，而是考在具体场景里应用知识。CS608不考"什么是BVA"，而是给你Climate.determine的specification让你填BVA表格；CS605不考"什么是NP"，而是给你CARA clique让你写certificate和verifier。你需要把每个抽象概念锚定到具体的输出格式上——表格列名、证明步骤顺序、代码分析结构——这些才是评卷人实际看的东西。只背定义就像背了菜谱但从未进过厨房：你知道食材但做不出菜。

题目语境怎么用：复习每个主题时问自己："这个概念在past paper里以什么形式出现？需要我画表、写证明还是做比较？"比如MVC概念在CS615考试中以"25-mark essay"形式出现，需要你写出请求流、职责分层和trade-off，而不是默写MVC三个字母。BVA概念需要你产出带TCI、selected value、expected result的完整表格。

卷面写法：先说知识对应的考试输出格式，再给一个past paper里的具体例子说明那种格式长什么样，最后解释为什么那种格式能得分（表格因为结构清晰、证明因为步骤完整、比较因为trade-off有依据）。

常见失分点：花大量时间背诵定义文字，但从未练习过把定义转换成考试答案格式。考场上看到题目认识概念，但不知道从哪开始写、写什么结构、写到什么程度算完。

核心答案（先确认概念）：遮住正文，用 30 秒说出本节的定义、一个例子、一个常见扣分点；说不出就回到本节重新读。

为什么这是对的：30秒限制是有意设计的——它迫使你提取最核心的信息，而不是试图复述所有文字。能说出定义证明你理解了概念边界（不是什么都知道，而是知道这个概念区别于其他概念的边界在哪）；能说一个例子证明你能应用（概念从抽象落到了具体）；能说出常见扣分点证明你达到了考试要求的精确度（知道哪里容易出错说明你理解了评卷标准）。三者都流畅的节可以放心过，任何一项卡住都说明那个维度有漏洞。

题目语境怎么用：学完CS603的Hoare logic节后，遮屏幕30秒说出：①Hoare triple的结构({P}S{Q}) ②一个具体例子（Find-min的invariant："min不大于已扫描区间a[0..i)中每个元素，且1<=i<=a.Length"）③常见扣分点（invariant缺范围无法支撑postcondition）。30秒到了检查自己漏了什么，然后决定是否重读。

为什么这是对的：Hoare logic证明题是CS603的必考题型，核心步骤固定但细节决定得分。Find方法证明的关键在于invariant的设计——"min不大于已扫描区间a[0..i)中每个元素，且1<=i<=a.Length"。这条invariant需要经过三步验证：初始化（i=1, min=a[0]时成立）、保持（每轮更新min和i后仍成立）、退出（i=a.Length时结合invariant推出min是整个数组的最小值）。流程固定，但invariant的语义来自对程序逻辑的理解——如果你不理解"为什么min绑定的是已扫描区间"，你只能背答案而不能独立写。

卷面写法：卷面写 precondition、invariant 初始化/保持/退出三步、postcondition 推导；若问 total correctness，再写 non-negative variant 每轮下降。

标准答案（先写结论）：min 不大于已扫描区间 a[0..i) 中每个元素，并且 1<=i<=a.Length。

为什么这是对的：这条invariant精确刻画了Find循环的中间状态，分两部分理解。前半句"min不大于a[0..i)"保证了min始终是已扫描区间的最小值——这是循环的核心不变量，每轮迭代要么保持min不变（新元素更大），要么更新min（新元素更小）。后半句"1<=i<=a.Length"限定了i的合法范围，防止数组越界，同时确保了退出条件(i=a.Length)时已扫描区间覆盖整个数组。两条合在一起，循环退出时直接推出"min不大于整个数组"即min是最小值，完美支撑postcondition。如果只写"min是最小值"而不限定i的范围和扫描区间的语义，初始化阶段就无法验证——因为循环开始前还没有"已扫描区间"。

常见失分点：不要只写 postcondition。Invariant 缺范围或缺语义，都无法支撑证明。

标准答案（先写结论）：搜 JaCoCo、branch coverage、short-circuit，再回到 Q2 题型页写 additional test + expected result + coverage explanation。

为什么这是对的：decideWrite是一道白盒覆盖率题。JaCoCo报告里的红色行表示statement从未执行——你需要构造一个输入让程序走到那行；黄色标记表示branch条件只覆盖了一侧——你需要构造输入让条件走向未覆盖的分支。short-circuit运算符(&&, ||)在字节码层面会产生额外的分支判断，JaCoCo会把它们一并标记出来，如果不知道这个机制就会困惑"为什么一个if有两个黄色标记"。理解了这三个概念，再看Q2题型页的worked example就能独立完成decideWrite的覆盖率补全。

题目语境怎么用：看到 line red/yellow，要问什么输入能到达这条 statement 或走到缺失 branch；short-circuit 条件还可能产生 bytecode-level branch。

卷面写法：卷面写 coverage item、input/call、expected result、补上的 line/branch。不要只给 input。

常见失分点：覆盖率不是正确性的替代品。每个新测试仍要有 specification 推出的 expected result。

卷面写法：先说两种结果类型分别解决什么问题（线性页=补理解，题型页=学写法），再说判断标准（根据当前是"不理解"还是"不会写"来选择），最后给一个关键词的双结果选择实例（如搜"Hoare"时线性页教{P}S{Q}语法，题型页教Find的invariant怎么写）。

常见失分点：每次都点题型页结果（因为看起来离考试更近），但概念基础不牢时看题型页只会更困惑。另一个错是每次两个都点，结果在理解和写法之间来回跳，两种都没吃透。

核心答案（先确认概念）：不能替代完整学习流程。它解决局部卡点，不负责建立整门课的顺序感。

为什么这是对的：一门课的知识有严格的层级结构。CS605里，不理解Turing Machine的定义就没法学HALT（因为HALT是关于TM停机的问题），没学HALT就没法学mapping reduction（因为reduction的目标就是把新问题归约到HALT）。关键词跳转可以在你学完Turing Machine后帮你快速回到那一节复习，但它不能帮你建立从TM到HALT到reduction的推理链条——这个链条只能通过按顺序走完线性页来建立，因为每步推理都依赖于上一步的概念。

题目语境怎么用：如果你对CS605的"decidability"整章都模糊——不知道Turing Machine的定义、不知道recognizable和decidable的区别、不知道为什么HALT不可判定——不要用关键词跳来跳去试图拼凑理解。回到CS605线性页，从Turing Machine那节开始按顺序重读，每一节都做遮屏复述。关键词只在你"大部分懂了，个别点忘了"的时候有效。

考试题目题（再做 5 题）

标准答案（先写结论）：CS605 Sample A Q3 exception 21 或 Sample B Q5 Java mean property 的 mapping reduction。

为什么这是对的：HALT归约题的通用逻辑是——假设存在判定目标性质的程序D，构造程序N先模拟M(w)：如果M(w)停机则N表现出目标性质（如抛exception 21、执行第n行），如果M(w)不停机则N不表现该性质。于是D(N)返回true当且仅当M(w)停机，与HALT的不可判定性矛盾。练这两道题的原因：Q3练习"停机时触发exception"模式，Q5练习"停机时执行指定代码行"模式，覆盖了最常见两种需要归约到HALT的性质类型。两道练完后你就掌握了"把任意程序性质变成停机开关"的构造思路。

题目语境怎么用：构造程序 N 时通常先模拟 M(w)：若停机，就触发 exception、执行指定 line 或破坏某个 throughout 条件；若不停机，就不触发该性质。

卷面写法：卷面四步：假设 decider D 存在；从 <M,w> 有效构造 N；运行 D(N)；证明 M(w) halts iff N 具有题目性质；最后写 contradiction。

常见失分点：不能只写 reduce to HALT。mapping 怎么造、为什么 computable、iff 两个方向都要写。

标准答案（先写结论）：CS608 2025 Climate.determine BVA，不包含 error values，并完成 TCI/data/test cases。

为什么这是对的：Climate.determine是BVA最标准的考试题型——从specification中提取阈值(16和60)，为每个阈值构造just below/on/above三组值，填入TCI表并推导expected result。练这道题的目的是建立BVA答题的肌肉记忆：读spec→圈阈值→对每个阈值取below/on/above→确认expected result来自spec哪句话。题目明确说"不包含error values"，意味着你只需要测有效输入空间的边界，不要把temp=-999或humidity=200%这种无效值混进去——这是很多学生扣分的地方。

题目语境怎么用：Climate 题围绕 temp=16、humidity=60；其他 BVA 题也要先找 threshold、minimum、maximum 或 between 的边界。

卷面写法：卷面用表格写 TCI、selected value、test case、expected result。每个 expected result 都必须来自 specification。

常见失分点：不要只写"测边界"。只列输入没有 expected result 也不完整；题目说不含 error values 时，别把无效值混入有效边界表。

标准答案（先写结论）：CS615 2025 MVC 25-mark essay 或 Sample Rails OrdersController SRP/refactor 题。

为什么这是对的：MVC在CS615考试中从不出"MVC是哪三个字母"这种填空题，而是以25-mark essay的形式出现——要求你在具体系统场景中解释请求流、职责分层和trade-off。练2025 MVC essay让你学会"从题面压力推出架构决策"的思维：题目给一个系统描述，你要分析谁接request（Controller）、谁执行业务规则（Model/Service）、谁渲染响应（View）、哪里有扩展或一致性压力（从而判断是否需要拆分或引入额外层）。练OrdersController refactor题让你学会"识别职责越界并重构"——比如Controller里写了太多业务逻辑应该抽到Service层。

常见失分点：不要只写"scalable"或"handles logic"。必须说明是哪种数据压力、职责边界或失败风险让这个选择合理。

标准答案（先写结论）：min 不大于已扫描区间 a[0..i) 中每个元素，并且 1<=i<=a.Length。

为什么这是对的：这条invariant精确刻画了Find循环的中间状态，分两部分理解。前半句"min不大于a[0..i)"保证了min始终是已扫描区间的最小值——这是循环的核心不变量，每轮迭代要么保持min不变（新元素更大），要么更新min（新元素更小）。后半句"1<=i<=a.Length"限定了i的合法范围，防止数组越界，同时确保了退出条件(i=a.Length)时已扫描区间覆盖整个数组。两条合在一起，循环退出时直接推出"min不大于整个数组"即min是最小值，完美支撑postcondition。如果只写"min是最小值"而不限定i的范围和扫描区间的语义，初始化阶段就无法验证——因为循环开始前还没有"已扫描区间"。
