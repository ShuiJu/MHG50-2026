复习系统怎么用

目标不是收藏很多材料，而是让每一次阅读都能落到考试动作上。四门课都按同一种节奏走：从零理解，马上接小题，再落到真题模板。

1. 用线性页铺台阶

每个新术语第一次出现时先解释，再给最小例子。适合从教材讲义顺序一路学完单科。

2. 用题型页压答案

把知识点转换成考试表格、证明模板、代码分析步骤和检查清单。适合做题和考前复盘。

3. 用关键词补薄弱点

不会哪个词就搜哪个词，例如 HALT、BVA、Dafny、MVC，直接跳到相关小节。

过关问答：先内容理解，再 past/sample paper 题，悬停或点击显示答案

题目语境怎么用：CARA clique、vertex cover、3-SAT clauses/literals 都是在给 certificate 和 verifier 的语境：clique 检查点集两两有 edge；3-SAT reduction 检查 clause/literal 到目标结构的对应。

卷面写法：in NP 写 certificate、verifier、polynomial time。NP-complete 写 membership，再从已知 NP-complete 问题构造并证明 yes iff yes。

常见失分点：不要把 NP-complete reduction 写成 HALT reduction。3-SAT reduction 用来证明 NP-hard；HALT reduction 用来证明 undecidable。

题目语境怎么用：看到 line red/yellow，要问什么输入能到达这条 statement 或走到缺失 branch；short-circuit 条件还可能产生 bytecode-level branch。

卷面写法：卷面写 coverage item、input/call、expected result、补上的 line/branch。不要只给 input。

常见失分点：覆盖率不是正确性的替代品。每个新测试仍要有 specification 推出的 expected result。

标准分析（为什么）：这道小题问的是“考试5. 首页考试补题：CS605 HALT 卡住时练哪两道？”。标准答案里的“Sample A Q3 exception 21 和 Sample A Q5 line number n never executed，二者都要写 HALT mapping reduction。”只是结论，真正要拿分的是说明题面为什么支持这个结论。 HALT 归约题的核心是把题目 property 变成停机问题的开关：如果目标性质可判定，就能判定 M(w) 是否停机，从而矛盾。

题目语境怎么用：构造程序 N 时通常先模拟 M(w)：若停机，就触发 exception、执行指定 line 或破坏某个 throughout 条件；若不停机，就不触发该性质。

卷面写法：卷面四步：假设 decider D 存在；从 有效构造 N；运行 D(N)；证明 M(w) halts iff N 具有题目性质；最后写 contradiction。

常见失分点：不能只写 reduce to HALT。mapping 怎么造、为什么 computable、iff 两个方向都要写。

每一节的过关闭环

流程最大的风险不是“没看材料”，而是看完就跳走，没有确认自己能不能说出来、写到卷子上。现在每个 section 底部都会出现 QA 和过关勾选，按下面四步走。

1. 读本节正文

只读到能用自己的话复述本节概念、例子和常见坑，不追求一次背完所有文字。

2. 做 3 道内容题

先遮住答案答内容理解题。能答出定义、关系、最小例子，才算内容地基过关。

3. 做 5 道考试题

再做 past/sample paper 导向题。答案要能说出题面对象、可交卷结构和具体例子。
