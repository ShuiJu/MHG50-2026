Section 1：几乎必出（CS605 固定 7 题格式）

CS605 考卷遵循固定的 7 题结构——每题 10 marks，题型从不偏离以下七个位置：Q1=not regular, Q1b=not CFL, Q2=decidable+T-r, Q3=undecidable reduction, Q4=TR/co-TR, Q5=second reduction, Q6=in NP ×2, Q7=NP-complete。即使语言定义换了，证明骨架完全不变。以下每题给出最可能出现的语言和完整标准答案。

Q1: Prove Language NOT Regular (10 marks)

题面预测：L = {w1#w2#...#w2n : wi ∈ {a}*, n > 0, exists odd i where wi=wi+1}

题目定位：Q1(a) — 使用 Regular Pumping Lemma 证明语言不是 regular。

标准答案（完整交卷版）：

Step 1 — Assume regular (反证法开头): Assume, for contradiction, that L is regular. By the pumping lemma, there exists a pumping length p > 0 such that for all strings w ∈ L with |w| ≥ p, there exists a split w = xyz satisfying (1) |xy| ≤ p, (2) |y| > 0, and (3) xyiz ∈ L for all i ≥ 0.

Step 2 — Choose witness word (选串): Choose w = ap#ap#ap#ap. This word has 4 unary blocks (n = 2, i = 1 is odd and w1 = w2 = ap ✓). Thus w ∈ L and |w| ≥ p.

Step 3 — Constrain y using |xy| ≤ p (锁定泵段位置): Since |xy| ≤ p, the substring xy lies entirely within the first ap block (before the first #). Therefore y consists only of a's from the first block, and |y| > 0.

Step 4 — Pump and break (破坏语言条件): Take i = 2. Then xy2z = ap+|y|#ap#ap#ap. The first block now has length p + |y| > p, while the second block still has length p. Thus w1 ≠ w2. Since w1 is the only odd-index block that originally equalled its neighbour, and all other block pairs have unequal lengths, the pumped word has no odd i with wi = wi+1. Hence xy2z ∉ L.

Step 5 — Contradiction (矛盾收尾): This contradicts the pumping lemma, which requires xyiz ∈ L for all i ≥ 0. Therefore, our assumption that L is regular is false. Thus L is NOT regular. ∎

得分关键总结：

  1. 必须写 "assume for contradiction" —— 这是反证法骨架的第一句，漏了阅卷人不知道你的假设。
  2. 必须用 |xy| ≤ p 锁死 y 的位置 —— 这是 pumping lemma 给你的最强武器。因为 |xy| ≤ p 且 w 的前 p 个字符全是最前面 ap 里的 a，所以 y 只由 a 组成，绝不可能跨越 #。
  3. 必须解释为什么 pumped word 违反语言条件 —— 不能只说 "not in L"；要说清楚它违反了哪个具体条件（wi = wi+1 被破坏）。
  4. 必须明确点出矛盾对象 —— "this contradicts the pumping lemma" 而不是泛泛的 "this is a contradiction"。

Q1b: Prove NOT Context-Free (10 marks)

题面预测：L = {u#v : u, v ∈ {a,b}*, |u| = |v|, u differs from v by exactly one symbol}

题目定位：Q1(b) — 使用 CFL Pumping Lemma 证明语言不是 context-free。

标准答案（完整交卷版）：

Step 1 — Assume CFL: Assume, for contradiction, that L is context-free. By the CFL pumping lemma, there exists a pumping length p > 0 such that for all strings w ∈ L with |w| ≥ p, there exists a split w = uvxyz satisfying (1) |vxy| ≤ p, (2) |vy| > 0, and (3) uvixyiz ∈ L for all i ≥ 0.

Step 2 — Choose witness word: Choose w = apbp#apbp-1a. Verify: |u| = p + p = 2p, |v| = p + (p-1) + 1 = 2p, so |u| = |v| ✓. u = apbp, v = apbp-1a — they differ at exactly one position (the last character of v is 'a' instead of 'b'), so w ∈ L. Also |w| ≥ p.

Step 3 — Case analysis on vxy (用 |vxy| ≤ p 分情况): Since |vxy| ≤ p, the substring vxy can span at most p characters. Consider all possible placements of vxy in w:

Casevxy locationWhy uv2xy2z ∉ L
Case 1vxy entirely within first ap (before first b)Pumping changes |u|'s a-count but leaves |v| unchanged → |u| ≠ |v| → not in L.
Case 2vxy entirely within first bpPumping changes |u|'s b-count → |u| ≠ |v| → not in L.
Case 3vxy spans the # separatorPumping with i = 0 removes # → the resulting string has no # → not in L (format violation). Pumping with i = 2 duplicates # → two #'s → not in L.
Case 4vxy entirely within second apPumping changes |v|'s a-count → |u| ≠ |v| → not in L.
Case 5vxy within second bp-1a regionPumping changes the tail of v — u and v now differ in more than one position or unequal length → not in L.
Case 6vxy crosses ap/bp boundary within u or vPumping disrupts the structure (e.g. creates ap+|vy|bp-|vy| if vxy spans a-b boundary) → |u| ≠ |v| or more than one difference → not in L.

In every possible case, uv2xy2z ∉ L (for cross-# case, i = 0 also works).

Step 4 — Contradiction: This contradicts the CFL pumping lemma. Therefore L is NOT context-free. ∎

得分关键：CFL pumping 必须用五段切法 uvxyz 而非三段 xyz。必须分情况讨论 vxy 的所有可能位置。|vxy| ≤ p 是分情况的依据——它保证 vxy 无法横跨超过 p 个字符的距离。

Q2: Prove DECIDABLE — FA language (10 marks)

题面预测：L = {<M> : M is FA with alphabet {0,1}, 011 ∈ L(M)}

题目定位：Q2(a) — 证明语言是 decidable。关键：FA 状态有限，保证搜索/模拟必然终止。

标准答案（完整交卷版）：

Decider D: "On input <M>:

  1. Validate encoding. Check that <M> is a valid encoding of a finite automaton. If not, reject.
  2. Simulate M on "011". Since the input word has exactly 3 characters ({0,1} alphabet), the FA reads them one by one, transitioning through states. Because an FA always halts on finite input (3 steps, each consuming one symbol), the simulation always terminates.
  3. Accept if M accepts, reject if M rejects. After reading all 3 symbols, M is either in an accepting state or not. If yes → accept <M>; if no → reject <M>.

Termination guarantee (decidable 的生命线): An FA on a fixed-length input string always halts — it reads exactly |w| symbols, each triggering exactly one deterministic transition. With 3 symbols, the simulation takes exactly 3 steps. Therefore D halts on every input. Hence L is DECIDABLE. ∎

得分关键：必须明确说明 "halts on every input"。证明 decidable 的最后一句永远是论证终止——如果忘记说，最高只能拿到 recognisable 的分数。FA 的有限性（finite states + finite input = finite simulation）是终止保证的来源。

Q2b: Prove Turing-RECOGNISABLE (10 marks)

题面预测：L = {<J,a,b> : J is Java program, at some point a and b are both zero simultaneously}

题目定位：Q2(b) — 证明语言是 Turing-recognisable。关键：simulate J，一旦条件满足就 accept；no instance 上可能永远循环。

标准答案（完整交卷版）：

Recogniser R: "On input <J, a, b>:

  1. Validate encoding. Check that <J, a, b> encodes a valid Java program with variables named a and b. If not, reject.
  2. Simulate J step by step. Maintain the current values of variables a and b. After each execution step (each bytecode or statement), check whether a == 0 and b == 0 simultaneously.
  3. If a == 0 and b == 0 at any step → accept. The recogniser immediately halts and accepts.
  4. If J terminates without ever satisfying a == 0 && b == 0 → reject.
  5. If J runs forever and never satisfies the condition → R runs forever. This is acceptable for a recogniser.

Why recognisable but NOT decidable: Determining whether a Java program EVER reaches a particular state (both variables zero) during its execution is undecidable by Rice's theorem — it is a non-trivial semantic property of the program's runtime behaviour. No algorithm can, in finite time, decide for all <J, a, b> whether J will eventually make both variables zero. The best we can do is simulate and hope — which is exactly what a recogniser does.

Therefore L is Turing-RECOGNISABLE. ∎

得分关键：必须明确区分 yes instance (accept) 和 no instance (可能 loop forever)。不能说 "always halts" —— 那样就声称了 decidable，会扣分。对比 Q2(a)：FA 题 decidable because finite states；Java/TM 题 recognisable because infinite state space / unbounded execution。

Q3: Prove UNDECIDABLE via Mapping Reduction (10 marks)

题面预测：L3 = {<J> : J is a Java program that takes no input, and J never throws Exception21}

题目定位：Q3 — 从 HALT (或 HALT complement) 做 mapping reduction 证明 undecidable。

标准答案（完整交卷版）：

Strategy: Reduce from HALTcomplement = {<M,w> : M does NOT halt on w}. HALTcomplement is known undecidable (if it were decidable, HALT would be decidable by flipping the answer).

Construct computable mapping f: f(<M,w>) = <N> where N is the following Java program (taking no input):

void main() {
    simulate M on input w;
    // If simulation ever halts:
    throw new Exception21();
}

N ignores its own input (takes no arguments), simulates M on w internally. If the simulation of M(w) terminates (M halts), N throws Exception21(). If M(w) runs forever, N's simulation runs forever and the exception is never thrown.

Prove iff — BOTH directions (双向证明):

DirectionProof
(⇒) <M,w> ∈ HALTcomplementM does NOT halt on w → N's simulation runs forever → N never throws Exception21 → <N> ∈ L3.
(⇐) <N> ∈ L3N never throws Exception21 → by construction of N, this can only happen if the simulation of M(w) never terminates (otherwise the throw would fire). Therefore M does not halt on w → <M,w> ∈ HALTcomplement.

Conclusion: <M,w> ∈ HALTcomplement iff f(<M,w>) = <N> ∈ L3. The mapping f is clearly computable (it just writes the code of N with M and w embedded). If L3 were decidable by some decider D, then we could decide HALTcomplement by: on input <M,w>, construct <N> = f(<M,w>), run D on <N>, and output D's answer. This would make HALTcomplement decidable — contradiction. Therefore L3 is UNDECIDABLE. ∎

得分关键：

  1. 必须声明 f 是 computable —— 归约函数必须是可有效计算的，不能依赖神谕。
  2. iff 必须两个方向都写 —— (⇒) 和 (⇐) 分别在卷面上明确标出，每个方向的推理链要完整。
  3. 归约方向不能反 —— 从已知 undecidable 问题 (HALT or HALTcomplement) 归约到目标 L3。方向是 HALTcomplement ≤m L3。
  4. 构造 N 的逻辑必须清晰 —— "先模拟 M(w)" → "模拟结束后触发/不触发 property" 是标准后置触发模式。

Q4: TR / co-TR Analysis (10 marks)

题面预测：针对 Q3 的语言 L3 = {<J> : J never throws Exception21}，分析它是 Turing-recognisable、co-Turing-recognisable、还是两者皆非。

题目定位：Q4 — 使用 "L and L̄ both T-r ⇒ L decidable" 的定理，结合 Q3 的 undecidable 结论。

标准答案（完整交卷版）：

Analysis of L3 = {<J> : J never throws Exception21}:

1. L3 complement (L̄3) = {<J> : J DOES throw Exception21}. This language IS Turing-recognisable. Reason: construct a recogniser that simulates J step by step. If at any point J throws Exception21, the recogniser catches it and accepts. If J never throws Exception21 (and loops forever), the recogniser runs forever — which is acceptable for a recogniser. Thus L̄3 ∈ T-r.

2. Is L3 itself Turing-recognisable? Use the theorem: if both L and L̄ are Turing-recognisable, then L is decidable. From Q3, we proved L3 is undecidable. Since L̄3 IS Turing-recognisable (proved above), L3 CANNOT be Turing-recognisable. Otherwise, if L3 were also T-r, running the recognisers for L3 and L̄3 in parallel would give a decider — contradiction to L3 being undecidable.

Conclusion: L̄3 is Turing-recognisable (T-r). L3 is co-Turing-recognisable (co-T-r). L3 is NOT Turing-recognisable itself. ∎

得分关键：

  1. 必须先确定 complement 是什么 —— L̄3 = "J throws Exception21"。
  2. 必须引用 Q3 的结论 —— L3 is undecidable (from Q3)。
  3. 必须用定理 "both T-r ⇒ decidable" —— 这是 Q4 的核心。因为 L3 undecidable，所以不能两边都是 T-r。一边已验证 T-r，因此另一边必定不是。
  4. 结论要说清楚哪个是 T-r、哪个是 co-T-r，不要搞反。

Q5: Second Undecidability Reduction (10 marks)

题面预测：L5 = {<J,n> : J is Java program, line n is never executed during any run of J}

题目定位：Q5 — 第二个 HALT mapping reduction，使用 "后置触发" 模式，将目标代码行放在模拟 M(w) 之后。

标准答案（完整交卷版）：

Strategy: Reduce from HALT = {<M,w> : M halts on w} (known undecidable).

Construct computable mapping f: f(<M,w>) = <N, n> where N is a Java program with a distinguished line number n, constructed as follows:

void main() {                        // line 1
    simulate M on input w;            // line 2
    /* line n — placed here */        // line n: int dummy = 0;
}

N ignores its own input. It first simulates M on w internally. The target line n is placed after the simulation. Therefore:

  • If M halts on w → simulation finishes → execution proceeds to line n → line n IS executed.
  • If M does NOT halt on w → simulation runs forever → execution never reaches line n → line n is NEVER executed.

Prove iff — BOTH directions:

DirectionProof
(⇒) <M,w> ∈ HALTM halts on w → N's simulation finishes → N executes line n → <N,n> ∉ L5 (the line IS executed, so NOT "never executed").
(⇐) <N,n> ∉ L5Line n is executed at some point → by N's structure, execution can only reach line n after the simulation of M(w) completes → M halts on w → <M,w> ∈ HALT.

Equivalently: <M,w> ∈ HALT iff <N,n> ∉ L5. This shows HALT ≤m L̄5. Since HALT is undecidable, L̄5 is undecidable, and therefore L5 is undecidable. ∎

得分关键：

  1. 目标行的位置是核心 —— 必须放在模拟之后。放在模拟之前，行总是执行，与 HALT 无关。
  2. 注意归约的 complement 方向 —— M halts ⇔ line n IS executed ⇔ <N,n> ∉ L5。所以映射到的是 L5 的补。这不影响证明——补也是 undecidable。
  3. iff 两个方向都要写清楚 —— 每个方向要说明 "N 的结构导致..." 的推理链。

Q6: Prove in NP — Two Sub-problems (10 marks × 2)

题面预测 Q6a: CARA clique — G = (V,E) student friendship graph, ∃ k students all pairwise friends.

题面预测 Q6b: Phone vertex cover — Phones = vertices, communication pairs = edges, ∃ k phones covering all pairs.

Q6a — CARA Clique ∈ NP (标准答案)：

Language: L6A = {<G,k> : G is an undirected graph of student friendships, and G contains a clique of size at least k}.

Certificate: A set S ⊆ V of k students.

Verifier V: "On input (<G,k>, S):

  1. Check size: Verify that |S| = k. If not, reject.
  2. Check membership: Verify that all vertices in S are in V. If not, reject.
  3. Check pairwise edges: For every pair of distinct vertices u, v ∈ S, check that (u,v) ∈ E. If any pair lacks an edge, reject.
  4. Accept if all checks pass.

Runtime analysis: Step 1: O(1). Step 2: O(k). Step 3: there are C(k,2) = k(k-1)/2 pairs. Each edge lookup is O(1) with an adjacency matrix, or O(log|V|) with a hash set. Total: O(k²) ≤ O(|V|²). Since the input size is polynomial in |V| + |E|, the verifier runs in polynomial time. Therefore L6A ∈ NP. ∎

Q6b — Phone Vertex Cover ∈ NP (标准答案)：

Language: L6B = {<G,k> : G = (V,E) where V = phones, E = communication pairs, and ∃ k phones such that every communication pair involves at least one of those phones}.

Certificate: A set S ⊆ V of k phones.

Verifier V: "On input (<G,k>, S):

  1. Check size: Verify that |S| = k.
  2. Check coverage: For each edge (u,v) ∈ E, check that u ∈ S or v ∈ S (at least one endpoint must be in S). If any edge has neither endpoint in S, reject.
  3. Accept if all edges are covered.

Runtime analysis: Step 1: O(1). Step 2: iterate over all |E| edges, each membership check O(1) using a hash set for S. Total: O(|E|) ≤ O(|V|²). Polynomial time. Therefore L6B ∈ NP. ∎

得分关键（两张证 in NP 通用）：三要素不可缺一 — (1) certificate 精确定义，(2) verifier 步骤逐条写，(3) 多项式时间分析。不能只说 "guess a solution and verify"。Vertex cover 的验证条件是 "至少一个端点在 S" 而非 "两个都在"——这块写反直接零分。

Q7: Prove NP-Complete — 3-SAT ≤p CARA Clique (10 marks)

题面预测：证明 CARA clique 是 NP-complete。已知 3-SAT is NP-complete，要求从 3-SAT 做多项式归约。

题目定位：Q7 — 两半证明：引用 Q6 的 NP membership + 从已知 NPC 做 polynomial reduction。

标准答案（完整交卷版）：

Part 1 — Membership in NP: By Q6a (or direct proof), L6A ∈ NP — certificate = set of k vertices, verifier checks pairwise edges in O(k²) time. This establishes the first requirement for NP-completeness.

Part 2 — NP-hardness: Reduction from 3-SAT to CARA Clique

Reduction construction (多项式时间): Given a 3-CNF formula φ with m clauses C1, C2, ..., Cm, where each clause Cj = (ℓj1 ∨ ℓj2 ∨ ℓj3) and each literal ℓ is either a variable x or its negation ¬x. Construct a graph G = (V,E) and integer k as follows:

  1. Vertices: For each literal occurrence in φ, create a vertex labelled by (literal, clause_number). There are exactly 3m vertices — 3 per clause.
  2. Edges: Connect two vertices (ℓi, Cp) and (ℓj, Cq) if and only if:
    • They are from different clauses (p ≠ q), AND
    • Their literals are NOT complementary — i.e., it is NOT the case that one is x and the other is ¬x for the same variable x.
  3. k: Set k = m (the number of clauses).

Polynomial size: |V| = 3m, |E| ≤ (3m)² = 9m². The construction is clearly polynomial in the size of φ.

Correctness proof — iff BOTH directions:

DirectionProof
(⇒) φ satisfiable ⇒ G has k-cliqueLet τ be a satisfying assignment. In each clause Cj, at least one literal is true under τ. Pick ONE such true literal per clause. The corresponding k = m vertices form a clique: (1) they come from different clauses, so the different-clause condition holds; (2) if two selected literals were complementary (x and ¬x), they couldn't both be true under τ — contradiction. So all edges exist.
(⇐) G has k-clique ⇒ φ satisfiableLet the k-clique be S. Since vertices from the SAME clause have NO edges between them (by construction — clause condition fails the "different clauses" requirement), the clique can contain at most one vertex per clause. With |S| = k = m clauses, the clique must contain exactly one vertex from each clause. Let τ be the assignment that makes all selected literals true (set x = true if literal is x, x = false if literal is ¬x). This assignment is consistent: if two selected literals were complementary (x and ¬x), there would be no edge between them by construction, contradicting that S is a clique. Thus every clause has at least one true literal — φ is satisfiable.

Conclusion: Since 3-SAT ≤p CARA Clique and 3-SAT is NP-complete, CARA Clique is NP-hard. Combined with membership in NP, CARA Clique is NP-COMPLETE. ∎

得分关键：

  1. 必须引用 Q6 的 NP membership —— "From Q6a, L6A ∈ NP." 这告诉阅卷人你知道 NP-complete 需要两半。
  2. 构造必须明确：每 clause 3 个 vertex；同一 clause 内不加边；不同 clause 的 vertex 仅在 literal 不冲突（非互补）时加边；k = clause 数。
  3. iff 两个方向都需要解释 clique 和 satisfying assignment 之间的对应 —— (⇒) 从赋值选 literal 构成 clique；(⇐) 从 clique 的 literal 读出赋值。
  4. (⇐) 方向要论证 "每 clause 恰好一个 vertex" —— 这需要用到 clause 内无边 + clique 大小 = clause 数。
  5. 多项式时间 —— 需要说明构造规模（|V| = 3m, |E| ≤ 9m²，多项式）。

Section 2：很有可能（语言定义换但证明骨架不变）

Section 1 的语言是最可能的版本。但如果考试换了语言定义，证明骨架完全不变——只是 witness word、构造程序的具体动作会变。以下是最可能的三种 "variant" 场景，以及标准答案的调整方式。

Alternative Q1: Not Regular — u,v same start symbol

题面预测（如果 Q1 不是 unary block）：L = {uv : u,v ∈ {0,1}*, |u| = |v|, u and v start with the same symbol}

标准答案骨架（witness word 变了，其他不变）：

Assume regular → pumping length p. Choose w = 0p1 0p1. Verify: |u| = p+1, |v| = p+1, both start with '0' ✓, so w ∈ L and |w| ≥ p.

Since |xy| ≤ p, xy is entirely within the first 0p block. Thus y consists only of 0's from u's prefix.

Pump with i = 2: xy2z = 0p+|y|1 0p1. Now u = 0p+|y|1 and v = 0p1. Since |u| ≠ |v|, the string ∉ L.

Contradiction. L is NOT regular. ∎

变体要点：witness word 的设计是关键——让 |xy| ≤ p 把 y 锁在前 p 个字符的单一区域，pump 后破坏 |u| = |v| 或起始符号条件。

Alternative Q3: TM language nonempty is undecidable

题面预测（如果 Q3 不考 Exception21）：L = {<M> : M is a TM, L(M) ≠ ∅} — 即 TM 是否接受至少一个字符串。

对比 FA version (Sample B Q2a): FA language nonempty 是 decidable（状态有限，图搜索终止）；TM language nonempty 是 undecidable（TM 可能无限循环，你无法在有限时间内确定 M 是否接受任何字符串）。

标准答案（从 HALT 归约）：

f(<M,w>) = <N> where N is a TM that, on any input x:

  1. Ignores its own input x.
  2. Simulates M on w.
  3. If the simulation of M(w) halts (whether accept or reject), Naccepts its input x.
  4. If M(w) never halts, N never halts (so N accepts nothing).

iff: M halts on w ⇔ N accepts at least one string (in fact, all strings) ⇔ L(N) ≠ ∅ ⇔ <N> ∈ L. Therefore HALT ≤m L. Since HALT is undecidable, L is undecidable. ∎

关键对比：为什么 Sample B Q2(a) 的 FA 版本是 decidable？因为 FA 状态有限 → 可达性搜索必然终止。TM 状态和纸带无界 → 没有搜索终止的保证 → 必须用 HALT 归约证 undecidable。这个对比是考试最爱考的区分点。

Alternative Q5: Throughout-execution property

题面预测（如果 Q5 不考 "line never executed"）：L = {<J,b,c> : J is Java program, float b, int[] c, throughout execution b < mean(c)}

关键区别：这是 "throughout execution"（贯穿运行全程）类的 property——程序运行的每一步都必须满足不变量。构造思路是：让 N 在模拟 M(w) 期间保持不变量，模拟停机后破坏不变量。

标准答案：

Construct f(<M,w>) = <N, b, c> where N is a Java program that:

  1. Initialises: Set b = 0.0, c = {1, 2, 3} — so that mean(c) = 2.0 and b < mean(c) holds (0.0 < 2.0).
  2. Simulates M on w: During simulation, N does not modify b or c — the invariant b < mean(c) is maintained throughout.
  3. After simulation halts: N deliberately sets b = 100.0 (or modifies c to {0}) — this breaks b < mean(c).

iff: M(w) does NOT halt ⇒ N's simulation runs forever, b < mean(c) maintained throughout ⇒ <N,b,c> ∈ L. M(w) halts ⇒ after halting, N breaks the invariant ⇒ <N,b,c> ∉ L. Therefore HALTcomplement ≤m L, so L is undecidable. ∎

"throughout" 类题的通用构造模板：(1) 在模拟前建立不变量（设置初始值使条件成立）；(2) 在模拟期间不碰不变量（让它自然成立）；(3) 模拟停机后故意破坏不变量。这样 M halts ⇔ 不变量被破坏 ⇔ 程序不满足 property。

Section 3：如果换考法——不同语言，同一模板

考试不会完全重复往年题。但如果题型位置被替换，证明结构完全不变——只是 witness word、构造程序的具体动作会变化。以下是最可能的三种 "换法"，以及如何用现有模板应对。

If Q1/Q1b swap — Different language, same pumping template

情况：Q1(a) 变成了 not CFL 而非 not regular，或者语言定义完全换了（如换成 {w#w : w ∈ {a,b}*} 或 {anbncn}）。

应对原则 — 模板不变，只换 witness：

模板步骤Regular Pumping (Q1a)CFL Pumping (Q1b)
假设Assume L is regularAssume L is context-free
切法w = xyz, |xy| ≤ p, |y| > 0w = uvxyz, |vxy| ≤ p, |vy| > 0
选 witness选 w 使 |xy| ≤ p 把 y 锁在单一字符区域选 w 使 |vxy| ≤ p 限制 vxy 只能覆盖局部（最多两个区域）
Pump 方式i = 0 或 i = 2，只重复/删除 yi = 0 或 i = 2，同时重复/删除 v 和 y
破坏什么数量相等、block 长度、格式三段等长、分隔符数量、格式

常见 witness 选择速查：

  • anbn (not regular): w = apbp — y locked in a's by |xy| ≤ p.
  • anbncn (not CFL): w = apbpcp — vxy can span at most 2 regions.
  • w#w (not CFL): w = apbp#apbp — vxy may cross #, pump with i = 0 to remove #.
  • ww (not CFL): w = apbpapbp — vxy may disrupt symmetry.

核心原则：无论语言定义怎么变，证明骨架永远是 "假设 → 泵长 → 选 witness → 约束泵段位置 → pump → 破坏条件 → 矛盾"。变的只是 witness word 的选择和 "为什么破坏" 的解释。

If Q2 swaps decidable / recognisable proof

情况：Q2(a) 和 Q2(b) 的题目类型互换——比如 (a) 变成 Java 程序题需要证 recognisable，(b) 变成 FA 题需要证 decidable。或者两个子题都是同一方向。

判定规则速查表：

输入对象条件类型结论证明方法
FA / DFA任何 property（accepts some word, word length > n, language empty, language infinite）DECIDABLE构造算法，利用 FA 有限状态论证搜索必然终止。例如：图搜索可达性、检查所有长度 ≤ n 的路径、BFS 标记状态。
PDALanguage emptiness, finitenessDECIDABLE将 PDA 转 CFG，检查 CFG 是否生成任何字符串（存在多项式算法）。
TM / Java program"eventually does X" / "at least once"Turing-RECOGNISABLE (not decidable)构造 simulator：模拟程序，一旦检测到条件满足就 accept。No instance 上 simulator 可能 loop forever。引用 Rice's theorem 说明不是 decidable。
TM / Java program"always does X" / "every run" / "throughout execution"Usually UNDECIDABLE (not even recognisable often)用 HALT reduction（从 HALT complement）证明 undecidable。通常构造程序在模拟 M(w) 时保持 property，模拟停机后破坏它。
TM / Java program"halts on all inputs", "accepts Σ*"UNDECIDABLE (not T-r)用 HALT reduction。Rice's theorem：any non-trivial semantic property of TM languages is undecidable。

核心区分：输入对象是有限对象（FA）→ decidable；输入对象是程序/TM + 条件涉及运行时行为 → recognisable (best case) 或 undecidable。FA 永远是 decidable 因为状态有限保证搜索终止。

If Q6/Q7 swap NP-complete source problem

情况：Q7 不再从 3-SAT 归约，而要求从 Vertex Cover、Set Cover 或其他已知 NPC 问题归约。Q6 的语言也可能不是 clique/vertex cover。

通用 in NP 证明模板（适用任何语言）：

  1. Certificate: 精确定义证书是什么数据结构——通常是一组顶点、一个赋值、一个子集、一个排列。
  2. Verifier steps: (a) 检查证书格式和大小，(b) 按语言定义逐条检查 yes 条件，(c) 全部通过则 accept。
  3. Runtime: 分析每步的复杂度，证明总体是多项式。常用：O(k²) 对选顶点集、O(|E|) 遍历边、O(n) 线性扫描。
  4. Conclusion: L ∈ NP.

通用 NP-hard reduction 模板（从已知 NPC 问题 A）：

  1. Reduction source: 明确声明从哪个已知 NP-complete 问题归约（如 "Reduce from 3-SAT, which is known NP-complete"）。
  2. Construction: 将 A 的每个实例部件（variable, clause, edge, vertex, set element）映射到目标问题的对应结构。每个 mapping 要说清楚。
  3. Polynomial: 论证构造的大小是原实例大小的多项式（如 |V| = 3m, |E| ≤ 9m²）。
  4. Correctness (⇒): A 的 yes instance → 构造出目标问题的 yes instance。从 A 的解构造目标解。
  5. Correctness (⇐): 目标问题的 yes instance → 原 A 的 yes instance。从目标解反推 A 的解。
  6. Conclusion: A ≤p L, so L is NP-hard. Combined with NP membership, L is NP-complete.

常见 NPC source problems 的构造方法速查：

FromTo核心构造思路
3-SATClique每 clause 3 个 literal vertex；不同 clause 的非互补 literal 连边；k = 从句数。
3-SATVertex Cover每个 variable 建 True/False gadget（两个 vertex 一条 edge）；每个 clause 建三角形；variable-clause 之间连边对应 literal 出现。k = |variables| + 2|clauses|。
3-SATIndependent Set和 Clique 互补：在 reduction graph 的 complement（补图）上找 independent set。
3-SATSet Cover / Hitting Set每个 clause 对应一个 set（包含其 3 个 literal）；universe = 所有 literal；找 hitting set 覆盖所有 clause。
CliqueVertex Cover直接 complement reduction：G has k-clique iff Ḡ (complement graph) has (|V|-k)-vertex cover。
Vertex CoverClique同上的对偶——互补图归约。

考试中，题目通常会明确告诉你 "You may assume 3-SAT / Vertex Cover / ... is NP-complete"，你只需要从这个指定问题出发做 reduction。不要自己凭空选 source problem——用题目给的。
