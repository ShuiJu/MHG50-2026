exam war room

考前最后一轮：把题面直接变成可交卷答案。

这一页不是重新学课。它专门训练考场动作：读题后先判断题型、确定要交付的产物、写出第一段、检查有没有漏掉能拿分的表格、公式、证明或例子。

基于 2023-2025 past papers 与 sample papers 题面触发词 最后三轮 整题包 漏分点

1. 读题分流：先判断要交付什么

考题的第一步不是回忆所有知识，而是识别这道题要你交付什么。CS615 常交付解释和取舍；CS605 交付证明；CS608 交付测试表和 expected result；CS603 交付公式、invariant、annotation 或 temporal property。

题面信号立刻归类必须交付第一检查点
shopping habits、receipt number、SQL/NoSQLCS615 database choice选择 + 收据系统例子 + trade-off有没有写 Customers/Receipts/ReceiptItems 和 transaction。
ReactJS、componentization、stackCS615 frontend/framework explanation定义 + component/props/state + stack 各层职责有没有只写“React 更快”。
Prove not regular / not CFLCS605 pumping proof选 word、任意 split、pump 后矛盾有没有覆盖所有合法切法。
Java program property、HALT、mapping reductionCS605 undecidability构造 N + iff + contradiction有没有说明 property 如何被 halting 控制。
BVA、TCI、test cases、without errorsCS608 black-box testingTCI table、data values、test cases、expected result有没有重复覆盖、有没有 expected output。
JaCoCo yellow/red、SC/BC、additional testsCS608 white-box testing缺口说明 + additional tests + coverage explanation有没有说明新测试补哪条 line/branch。
Hoare logic、partial/total correctnessCS603 program proofpre/post、invariant、rule use、variant if total有没有把 invariant 和 exit condition 接到 postcondition。
Spin、NuSMV、LTL、CTL、temporalCS603 model checkingstate model + formula + verification/counterexample有没有给具体 formula，例如 G(request -> F response)。

2. 最后三轮复习法

最后一轮不要平均用力。按题目产物复习，比按章节标题复习更接近试卷。每一轮都要写一点东西，不能只看。

第一轮：整题识别

拿一份 past/sample paper，只看题面关键词，不写完整答案。给每题标出：解释题、比较题、证明题、测试表、代码覆盖、Dafny、model checking。

第二轮：答案骨架

每题只写 5 行骨架：定义、题面对象、交付产物、例子、最后检查点。目标是速度和方向，不追求完整段落。

第三轮：补高频漏分

回看自己漏掉的内容：CS605 的 iff、CS608 的 expected result、CS603 的 variant、CS615 的 trade-off。只补这些会直接丢分的点。

推进规则：一道题如果 60 秒内说不出“题型 + 交付产物 + 一个具体例子”，就回到对应题型页；如果能说出但写不细，再回线性页补该 section。

下面不是新题，而是从 past/sample papers 里挑出的高回报训练包。每个包都覆盖一门课的主要得分动作。

4. 开头句：先把方向写正

开头句不是背范文，而是防止第一段写偏。先写对方向，后面才有空间展开。

题型可直接使用的开头下一句必须接什么
SQL / NoSQLFor this scenario I would choose SQL because the data is structured purchase/receipt data rather than independent event logs.接 Customers、Receipts、ReceiptItems、receipt number、transaction。
Microservices vs monolithThe choice is a trade-off between independent scalability and operational complexity.接 FinTrack 的 real-time prices、transaction history、team/deployment risk。
Pumping proofAssume for contradiction that the language is regular. Let p be the pumping length.接自己选择的 w，并说明任意合法 split 的限制。
HALT reductionAssume the target language is decidable. I will use its decider to decide HALT.接从 <M,w> 构造 N 的步骤。
BVAThe key boundaries in the specification are ...接 just below/on/above 的具体值和 expected result。
JaCoCo additional testsThe current suite misses the following coverage items...接 red/yellow line、触发条件、test case、expected result。
Hoare loop proofThe loop invariant should describe the part of the data already processed.接 establishment、preservation、exit use；total correctness 再接 variant。
Model checkingThe system is represented as states and transitions, and the requirement is represented as a temporal formula.接具体 LTL/CTL 公式和 counterexample 的意义。

5. 最常见漏分点

这些不是小错误，是真题里经常让答案从“懂”掉到“不给分”的位置。

科目危险写法改成这样
CS615只列 React、Node、SQL、NoSQL 名词。写每层职责、request flow、数据一致性、安全/部署 trade-off。
CS615Security 只写 sanitize。SQL injection 写 parameterized queries；XSS 写 output encoding、sanitization、CSP、HttpOnly cookies。
CS605Pumping proof 只展示一个切法。写任意合法 split，并用条件限制被 pump 的段在哪里。
CS605Reduction 只写“reduce from HALT”。写构造、computable mapping、iff、contradiction。
CS608Test case 只有输入，没有 expected result。每个 test case 都写 call、inputs、expected output，并说明覆盖哪个 TCI。
CS608White-box 只说“add tests”。指出未覆盖 line/branch，给触发条件和 expected result。
CS603Invariant 只写自然语言“it works”。写范围 invariant、已处理部分性质、退出时如何推出 postcondition。
CS603LTL/CTL 只写工具名。写状态模型、公式、operator 含义和 counterexample 解释。
