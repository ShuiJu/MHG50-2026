CS608 linear course path

CS608 全教材线性学习：从测试思想到随机和 AI 辅助测试

这页按 01-26 讲义顺序学习。先理解测试为什么需要设计，再学黑盒、白盒、对象上下文、自动化、随机测试和软件过程。

1. 测试是什么，为什么不能只靠感觉

测试的目标不是证明程序完全没错，而是在有限时间内系统地寻找重要错误。先学 specification、test case、expected result、coverage。

顺序内容材料过关标准
1.1Module overview01-module-overview.pdf知道四大考试题型。
1.2Intro to testing02-intro-to-testing.pdf能解释测试设计、expected results、coverage。

2. 黑盒测试：从规格选择输入

黑盒测试不看代码，只看规格。先学 EP，再学 BVA，最后学 DT。EP 问“哪些值行为相同”；BVA 问“边界会不会写错”；DT 问“组合会不会写错”。

顺序内容材料过关标准
2.1Equivalence Partitioning03-ep.pdf, 05-ep-more-detail.pdf能列 input/output partitions、TCI、test cases。
2.2Boundary Value Analysis06-bva.pdf, 07-bva-more-detail.pdf能找边界值并避免测所有组合。
2.3Decision Tables08-dt.pdf, 09-dt-more-detail.pdf能列 causes/effects/rules/infeasible combinations。

3. 白盒测试：从代码覆盖补遗漏

黑盒测试后，用 JaCoCo 看哪些语句和分支没执行。SC 关注语句，BC 关注条件结果。

顺序内容材料过关标准
3.1Statement and Branch Coverage10-sc-bc.pdf能解释 SC/BC 和 additional tests。
3.2Coverage details11-sc-bc-more-detail.pdf能读 JaCoCo yellow/red lines、short-circuit branches。

4. 对象和类内上下文

对象方法依赖对象状态。类内测试要创建对象、设置状态、调用方法、读取结果。

顺序内容材料过关标准
4.1OO basics12-oo-1.pdf知道 class、object、attribute、method。
4.2Testing in class context13-oo-further-look-at-class-context.pdf能写 setter -> method -> getter -> assert。
4.3OO issues and advanced14-oo-issues.pdf, 15-oo-advanced.pdf, 16-oo-more-detail.pdf知道对象上下文会影响测试设计。

5. 应用、自动化和工具

测试设计需要能落到自动化执行。理解 TestNG、JaCoCo、工具链、自动化结构。

顺序内容材料过关标准
5.1Application testing17-app.pdf, 18-app-more-detail.pdf知道测试从 unit 延伸到应用层。
5.2Automation19-auto.pdf能写参数化测试思路。
5.3Tools20-tools.pdf知道工具帮忙执行和测 coverage。

6. 随机、数值、AI 和过程

后半段把测试扩展到随机数据、数值问题、AI 辅助和软件过程。随机测试仍然要依赖人工设计的 TCI。

顺序内容材料过关标准
6.1Black/white and full testing21-bandw-full.pdf能解释为什么先黑盒再白盒。
6.2Random testing22-rand.pdf能说明 data/oracle/completion 三问题。
6.3Numerical computing23-NumericalComputing.pdf, floating point paper知道浮点误差会影响测试。
6.4Using AI and process24-using-ai.pdf, 25-process.pdf, 26-wrapup.pdf知道 AI/过程只是辅助，测试设计仍需可解释。

7. 最后接到考试

读完线性页后，回到 CS608 题型页 做四大题，再用 Q1 样章精练 BVA/DT。
